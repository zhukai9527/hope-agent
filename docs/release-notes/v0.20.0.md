# Hope Agent 0.20.0 — 设计空间与产物交付闭环

> **简体中文** · [English](https://github.com/shiwenwen/hope-agent/blob/v0.20.0/docs/release-notes/v0.20.0.en.md)

> 发布日期：2026-07-19

> 详见 [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.20.0/CHANGELOG.md#0200---2026-07-19)。

v0.20.0 从 v0.17.0 直接跳版，累积了原定 0.18、0.19 两个版本的改动，共 58 个提交。本版的主线是全新的**设计空间（Design Space）**——一个 Agent 原生的设计工作空间，从一句话需求直接产出网页、移动原型、演示文稿、海报、交互组件与音频等可交付产物，并可关联真实代码仓库，把设计稿一键落成组件代码。围绕这条主线，本版引入了本地优先的 **Artifacts 产物平台**与结构化数据分析报告，新增由 `/goal`、`/mode`、`/workflow`、`/loop` 构成的 **Agent 控制平面**，把图片与音频生成统一收敛为「服务商 → 多模型 → 功能默认链」体系，将记忆系统升级为 **Memory UX v2**，并在工作台补齐完整的 Git 与 PR 控制面。文件、附件与上传能力也在本版统一，本地桌面、远程桌面与 Web 三种形态下行为一致。

## 亮点

- **全新「设计空间」子系统**：侧边栏「知识空间」下方新增独立入口，输入一句话即可边生成边预览，直接产出自包含的设计产物——网页、移动原型、演示文稿、仪表盘、海报、文档、邮件、图像、动效、音频与真 React 交互组件。需求说不清时 AI 会主动追问受众与用途，需要定视觉方向时会弹出带配色与字体样张的风格卡；内置 6 套原创设计系统与一批主流品牌风格参考可一键换皮，也可从截图、网址、现有代码库或 Figma 文件反向提取设计系统。产物支持点选元素可视化微调、双击文字就地编辑、批注钉与画框批注，并可导出 HTML / PNG / PDF / PPTX / MP4 / ZIP 或部署到 Cloudflare Pages。
- **设计稿与代码仓库双向打通**：设计项目可关联一个真实代码仓库，产物菜单新增「实现到代码…」，把设计稿连同实现要求交给主对话，在你既有的技术栈里落成组件代码，每一步文件改动都走正常审批与 Diff 面板。反向同样成立：之后若在仓库里改动了这些落地文件，设计空间会自动察觉并在对应产物上提示，可查看差异、让 AI 据最新代码同步设计稿，或标为已同步。
- **新增 Agent 控制平面：目标、执行模式、动态工作流与循环**：新增 `/goal`（声明最终目标与必要完成标准，并按证据审计达成情况）、`/mode`（设定持续生效的推进强度）、`/workflow`（把一次任务编排成可预览、可审批、可观察、中断后可恢复的动态执行脚本）、`/loop`（按固定间隔或事件重复触发，并带无进展守卫）四条斜杠命令与配套工具。右侧工作台新增统一控制面板，可查看目标进度与证据、工作流运行时间线与审批记录、循环运行历史、代码审查与验证结果。
- **图片与音频生成统一为服务商体系**：图 / 音生成能力改为「服务商 → 多模型 → 功能默认链」统一配置，服务商与凭据在模型配置页集中管理，内置模板从 9 个扩到 27 个，覆盖 OpenAI、Google、火山方舟、腾讯混元、Fal、MiniMax、ElevenLabs、Black Forest Labs 等主流厂商，并支持自定义 OpenAI 兼容端点。新增聊天工具 `audio_generate`，可在对话内直接生成语音、音乐与音效。
- **本地优先的 Artifacts 产物平台与原生数据分析**：在既有 Canvas 之上加入统一的产物身份、不可变版本、内容寻址、版本恢复与来源证据摘要，全部在本机完成，无需公网部署；旧 Canvas 项目原样兼容。同时新增 Hope 原生数据分析能力，可处理 CSV / XLSX、项目文件与知识空间资料，产出带指标口径、发现、图表、数据质量与来源审计的结构化报告，关键数字必须可重算，数据不足时明确标注而非编造。
- **记忆系统升级为 Memory UX v2**：全局、Agent 与项目三层核心记忆统一以稳定快照进入 Prompt，详细主题留在动态存储中按需读取；新增核心记忆管理、待确认记忆审核与四档学习模式。自动动态召回改为明确 opt-in——默认只使用有界的核心记忆，开启后先跑零额外开销的确定性快召回，更耗时的深度召回独立默认关闭。
- **工作台补齐完整 Git 与 PR 控制面**：Git 状态、运行位置与托管工作树生命周期收口到同一张控制卡，可查看 staged / unstaged 差异，按全部、文件或 hunk 暂存、取消暂存与撤销，切换分支，并完成提交、推送与创建 draft PR。关联 PR 使用独立右侧面板，展示评审意见、checks 明细与未解决的行内评论，可把单项或全部反馈填入当前会话修复。

## 升级提示

本版有两处配置不做自动迁移，升级后需要手动重新设置：

- **图片 / 音频生成配置不迁移**：旧「设置 → 工具 → 图片生成 / 音频生成」的配置不会自动转换到新的服务商体系，升级后需要在「设置 → 模型配置 → 媒体生成模型」重新添加服务商与凭据，并在「设置 → 工具 → 媒体生成」确认四条功能默认链。
- **项目指令不迁移**：项目指令由数据库改为存放在项目根目录的 `AGENTS.md` 文件，原有内容不会被迁移，升级后需要在 `AGENTS.md` 中重新填写。

## 文件、附件与产物

- **文件与附件能力统一覆盖本地桌面、远程桌面和 Web**：文件树、消息附件、输入框草稿、工具媒体、工作台产物与 Artifact 统一使用同一套文件类型、预览、打开、下载、编辑和右键动作；本地桌面操作当前电脑，远程桌面与 Web 操作 Server 文件系统，未发送的附件始终留在客户端并可离线预览或编辑副本。
- **文件大小限制可配置并统一使用分块上传租约**：设置中新增聊天附件、工作台上传、文本预览 / 编辑、文档预览、Artifact 导入与知识来源限制。客户端文件改为 4 MiB 分块、可续传的临时租约，支持并发上限、失败回收与完成时校验，Web 与远程桌面不再一次性缓冲完整文件。
- **产物库导航、布局与状态标签统一**：侧边栏入口移动到数据大盘下方并统一命名为「产物库」，产物列表与属性面板支持折叠与拖拽调宽并记住偏好；类型、隐私、分析 / 验证状态等不再暴露英文 snake_case。本地导出不再要求填写「预期接收者」，只有敏感来源或可执行内容才在导出当下提示一次风险确认。

## 对话与协作

- **子 Agent 内联胶囊与独立「子智能体」面板**：对话里的子 Agent 由整块卡片改为紧凑胶囊，点开进入右侧「子智能体」面板，可按运行中 / 已完成分组浏览本会话全部子 Agent、查看完整调用信息，并与子会话实时对话、逐层下钻嵌套子 Agent。
- **对话消息支持选中引用到输入框**：在用户消息或助手回复中选中文字后右键，可复制选区或「添加到对话」；引用以可删除卡片暂存，支持连续添加与恢复编辑。
- **聊天消息排队改为持久、可恢复队列**：回复生成期间继续发送的消息会立即连同附件写入本地数据库，并在输入框上方展示真实队列状态；应用重启、页面刷新与多客户端并发时可恢复且不会重复发送。
- **浏览器 / macOS 控制面板支持悬浮小窗与执行历史**：两个实时镜像面板可切换为应用内悬浮小窗，停靠态底部新增执行历史时间线、会话统计与快捷操作条。
- **未读提示口径统一为「未读会话数」**：一个会话无论积压多少条回复都只计一次；只有窗口处于前台、界面可见且消息列表停在最新位置时，当前会话才算已读，切走或上翻历史时后台到达的回复不再被静默标记已读。

## 模型与服务商

- **内置 Provider 模板刷新**：新增 Claude Sonnet 5 与 Mythos 5、GPT-5.6、Kimi K3、Grok 4.5、Qwen 3.7 Plus 等当代旗舰，并新增 Cohere、Baseten、LongCat、Meta、Featherless 五个 Provider，内置模板增至 50 个 Provider、392 个预设模型。
- **Provider 支持声明单价币种**：新增「计价货币」（USD / CNY），单价照厂商价目页原文录入、成本大盘统一换算成 USD 统计，人民币价目不再被当作美元入账。
- **会话模型选择改为会话独立**：在输入框模型选择器中切换模型、思考强度或温度只影响当前会话，需要时可显式「同时设为 Agent 默认」；首选模型所在服务商被停用或删除后不再被静默改写，而是保留原选择并在选择器顶部提示。
- **后台一次性 AI 任务支持独立模型配置与自动降级**：Recap、Dreaming、知识空间整理、技能自动评审、图片 OCR、精灵模式、笔记工具与记忆召回摘要等后台任务可统一配置一条默认模型链，也可各自覆盖，并在所选模型不可用时自动尝试链上的下一个。
- **上下文计量与延迟工具加载统一为 Provider-aware 契约**：`/context`、压缩判断、缓存读写与数据大盘统一使用同一实际请求口径；工具目录拆分为 eager / deferred / activated，`tool_search` 发现后会真正激活工具并在下一轮可调用。

## 数据大盘与设置

- **新增「目标与执行」统一推进页**：集中展示 Goal 达成与必要标准、Workflow 完成 / 失败 / 审批、Loop 强推进与无进展守卫、Task/Plan 进度与当前需处理项，支持时间、Agent 与项目筛选，风险项可跳回会话定位。
- **成本改按你配置的单价结算**：数据大盘此前无视 Provider 编辑页中逐模型填写的单价，一律按内置估算表计算；现在优先按「服务商 + 模型」取你配置的单价，同一模型走不同渠道会分别结算。模型单价同时区分「未标价」与「明确免费」，本地模型与订阅制端点如实记为 $0，不再被按估算价收费。
- **用量支持按用途拆分**：Token 用量趋势新增按功能大类与精确用途标签两级拆分，可下钻到对应的 token 与成本明细。
- **设置页支持分层恢复默认值**：各设置页、页签与单个分区都有各自的「恢复默认值」入口，恢复前二次确认并说明影响范围；Provider、凭据以及已有记忆、知识空间、任务与产物不会被删除。
- **首次引导精简**：初次配置由 11 步收敛到 6 步，连接远程服务器改为欢迎页上的可展开次级入口；升级前尚未走完引导的用户会自动续到最近的剩余步骤。

## 界面与交互

- **全局焦点反馈与表单控件视觉统一**：鼠标或触摸操作不再出现突兀的深色焦点框，键盘导航保留轻量提示，并新增可手动开启的增强焦点模式；搜索框、输入框、选择器与数字输入同步改为更扁平克制的统一表面。
- **右侧面板入口重做为统一停靠栏**：标题栏上原本各自为政的「文件」「工作台」「后台任务」按钮与面板切换器合并为一条停靠栏，所有可用面板以同一排图标呈现。
- **对话输入区浮层交互统一**：「+」菜单、权限与沙箱设置统一采用模型选择器的浮层样式与动画，沙箱设置整合进权限弹层并默认折叠。
- **列表条目交互反馈统一**：知识空间与定时任务的悬停 / 选中态统一使用中性色阶，不再以浅蓝背景或蓝色文字表达普通选中；状态、错误、未读与拖拽落点仍保留独立语义颜色。
- **面板拖拽调宽更跟手**：聊天侧边栏、右侧面板、文件浏览器与知识空间的拖拽分隔线扩大了可拖动区域并改为高亮面板边框，拖动时面板严格跟随光标、不再有动画拖尾。

## 修复与稳定性

- **记忆读写不再卡死或拖住对话**：修复保存 / 更新 / 搜索记忆时因嵌入向量缓存与数据库连接相互等待而彻底卡住的问题；记忆与知识的自动召回改为在后台线程执行并加上超时与并发上限，慢磁盘或大量记忆时最多降级为本轮少注入几条参考。
- **审批与提问的倒计时、停止与撤窗不再失准**：审批弹窗与提问卡片的倒计时改用后端权威截止时间，页面重载或应用重启后依然准确；点击「停止」或删除会话时，在途的审批与提问会被立即拒决并在桌面 / Web / IM 各端同时撤窗。
- **定时任务里的循环任务状态与标识修正**：由 `/loop` 创建的循环任务不再沿用底层触发器状态，已结束的循环显示为「已完成」而非误报为可恢复的「已暂停」（升级后启动时自动修正历史数据），任务名不再外泄内部的 `[Loop] ` 前缀。
- **修正多家模型的定价与上下文长度**：Claude Opus 4.8 / 4.7 / 4.6 此前按旧档位记为 $15/$75（实际 $5/$25），Opus 4.7 / 4.6 与 Sonnet 4.6 的上下文仍写 200K（实际 1M）——上下文写大会让压缩迟迟不触发直至撞满真实上限报错。DeepSeek、Mistral Small、GPT-5.6 系列的上下文与价格一并校正。
- **界面本地化全面补齐**：数据大盘、知识空间、工作台、团队与各设置面板中残留的英文硬编码文案与直出的英文状态值已改为 12 种语言的本地化文案；设置与 Agent 能力页中约 60 个较新工具的名称与描述也补齐了全部语言。
- **macOS 点击交互更可靠**：非活动窗口中的主窗口、快捷对话、Plan、文件浏览与 Canvas 面板可直接响应首次点击；同时兼容部分输入法下 WebKit 偶发颠倒 `pointerup` / `pointerdown` 顺序的问题，避免控件需要点击两次。
- **Git Handoff 与提交失败恢复更安全**：Worktree 目标严格限制为当前会话或其子会话所有，失败回滚只撤销快照中的改动并保留无关文件；commit 成功但可选 push 失败时会保留提交并返回明确警告，不再误报整次操作失败。
- **修复旧版 Homebrew 下 Cask 安装被误拒**：cask 的 macOS 版本要求改为「不低于 Big Sur」的兼容写法，此前旧版 Homebrew 会理解成「仅限 Big Sur 这一个版本」而导致安装失败。
- **应用内全屏面板支持统一双向过渡**：Canvas、文件浏览器、单文件预览、Plan 与产物阅读器都会从当前布局位置平滑铺满窗口并在恢复时收回；系统开启「减少动态效果」时自动直接切换。
