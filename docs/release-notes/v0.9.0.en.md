# Hope Agent 0.9.0 — A Smoother Auto-Update Experience

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.9.0/docs/release-notes/v0.9.0.md) · **English**

> Released: 2026-06-09

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.9.0/CHANGELOG.md#090---2026-06-09) for details.

v0.9.0 turns auto-update into a hands-off yet resilient pipeline: both the desktop app and the server daemon surface new versions in the background and can pre-download them silently, you stay in control of when to restart, and the upgrade itself is far more tolerant of flaky networks and broken builds.

## Highlights

- **Hands-off updates**: the desktop app and the headless daemon both check for new versions in the background on a configurable interval and can pre-download them silently (on by default), so installing is near-instant. A new **Settings → About → Auto-update** section lets you toggle background checks, the check interval, background download, and update notifications independently; opening the About page also runs a check, and the "Check for updates" entry point is more prominent.
- **Choose how to restart**: when a new version is found you can pick "Update & restart" or "Update only (restart later)" — the app no longer restarts automatically and interrupts your in-progress conversation. You decide when to restart.
- **Resilient self-update**: downloads now retry on failure and resume via HTTP range, so they survive flaky networks; after swapping the binary the new version is smoke-tested on a cold start, and if it fails to launch the previous version is automatically rolled back. Leftover download temp files are cleaned up.

## Fixes

- **macOS camera / contacts / calendar / reminders / photos work in signed builds**: enabling Hardened Runtime added the corresponding resource-access entitlements, so these permissions are no longer blocked by the system after you grant them.
- **Correct self-update path for Homebrew installs**: a macOS build installed into the Applications folder via a Homebrew cask is no longer mistaken for a manual drag-install — upgrades correctly go through `brew upgrade` and stay consistent with Homebrew's install records.
