# Hope Agent 0.16.0 — Knowledge Space: Sources → Notes

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.16.0/docs/release-notes/v0.16.0.md) · **English**

> Released: 2026-07-06

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.16.0/CHANGELOG.md#0160---2026-07-06) for the full list.

v0.16.0 centers on a complete **Knowledge Space** "sources → notes" workflow: archive learning materials — web pages, chat media, local documents — as *sources*, then have an Agent organize them into reviewable, traceable Markdown notes. Around that headline, this release also brings drag-to-reorder projects in the sidebar, `@Agent` delegation from the input box, and a batch of improvements to project management, responsive UI, and MCP / embedding stability.

## Highlights

- **Knowledge Space gains a "sources → notes" compile workflow**: you can now archive learning materials — web pages, images / audio / video from chats, local documents — as Knowledge Space *sources*. After importing them via browser capture, image OCR, or speech-to-text, a background pass organizes them into reviewable Markdown note proposals — inspect the diff before anything is written, and the resulting notes can be traced back to their source Evidence. Sources support refresh and versioning, sync with external vaults, and governance over similar / duplicate sources. A new "source-organizing Agent" setting lets you dedicate a stronger Agent to this sources-to-notes work; it defaults to your global default Agent and does not affect your chat or recap model setup.
- **Drag-to-reorder projects in the sidebar**: you can now drag active projects in the conversation list to manually arrange their order, persisted across restarts, instead of relying on update-time ordering; starting a drag collapses any expanded project session groups so whole rows reorder cleanly.
- **`@Agent` delegation from the input box**: typing `@` in the conversation input box now lists Agent candidates after files / knowledge notes / skills; selecting one inserts a visual Agent chip and, on send, injects the intent "have this Agent handle the task" to the model so it can delegate through the existing subagent tools.
- **Refreshed built-in models and speech-to-text**: added the latest flagship models across Anthropic / OpenAI / Google Gemini / Moonshot / Zhipu / MiniMax (Claude Fable 5, GPT-5.6, Gemini 3.5 Flash, GLM-5.2, MiniMax M3, and more) plus an OpenCode Zen gateway template; speech-to-text adds ElevenLabs Scribe and xAI Grok STT cloud backends, Groq / Mistral Voxtral / DeepInfra OpenAI-compatible presets, and mxbai-embed-large in the local embedding catalog.

## Projects & UI

- **Project working-directory picker can create new directories**: when creating / editing a project, the working-directory picker moves to the top of the form and lets you pick an existing directory or create a new one; if you make no explicit choice, the default project-directory auto-creation still applies. The remote directory picker in HTTP Server mode also supports creating directories and remains gated by the `filesystem.allowRemoteWrites` toggle.
- **Project icons unified to logo / default folder**: creating / editing a project drops the Emoji field, so the project list, quick commands, cron jobs, and system prompts all identify a project by its logo or the default folder icon; projects without a logo also get their unread badge back.
- **More compact sidebar, unified main-window minimum height**: the icon sidebar now hides its bottom settings shortcuts when it gets too short (still keeping the divider between the function area and the status / log area), and the Plan history entry moves directly above Logs; the main window's minimum height is unified to 520 so the Canvas / Plan panels no longer squeeze the window shorter.
- **Responsive input toolbar and model-name hint**: as the input box narrows, it progressively folds the sandbox and permission-mode controls into the "+" menu (expandable in place), while the Stop / Send button stays on one line — at the narrowest, only "plus · model selector · Stop / Send" remain; when a long model name is truncated in the bottom toolbar, hovering shows "provider · full model name".

## Fixes & stability

- **MCP SSE transport works again**: `kind: sse` MCP servers are no longer incorrectly connected as Streamable HTTP (which failed with `400 session_id is required`) and now complete the proper SSE handshake; connections to local addresses (`localhost` / private networks) automatically bypass the system proxy, so a local MCP is no longer hijacked by the proxy while you route cloud models through it.
- **MCP config editing and hot-reload are more reliable**: editing an MCP server no longer writes the `<redacted>` placeholder back over real credentials, and the server name and id stay immutable; disable / reorder / global-toggle changes sync to the runtime immediately, sanitized tool-name collisions no longer overwrite each other, and concurrently cold-starting the same server no longer double-handshakes.
- **The skill can read the configured embedding**: `get_settings("embedding")` in the `ha-settings` skill no longer returns all-null and instead reflects the embedding model actually enabled in settings (with the API key redacted); embedding writes are now managed solely through the settings UI.
- **Copy only the selected text in a message**: after selecting part of the text in an AI reply, right-click copy and `Ctrl/Cmd+C` now copy just the selection instead of the whole message; with no selection, the right-click menu still copies the entire message.
- **More accurate sticky headers and elapsed times**: the hovering sticky user-message header now switches to match the currently visible reply and correctly handles collapsed rows, context-compaction events, and long-session DOM trimming; tool / thinking elapsed times no longer double-count the same round.
