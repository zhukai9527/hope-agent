# Hope Agent 0.1.1 — Maintenance Release

> [简体中文](./v0.1.1.md) · **English**

> Release date: 2026-05-09
> [`CHANGELOG.md`](../../CHANGELOG.md#011---2026-05-09) for the full list.

A maintenance release. The headline fix is the missing-DLL crash (`MSVCP140_1.dll` / `VCRUNTIME140.dll`) that v0.1.0 hit on clean Windows machines, plus a handful of independent fixes and one internal-naming change. No new features — the cross-device, cross-endpoint experience is unchanged from v0.1.0.

## Headline Fixes

- **Windows missing-DLL crash on launch** — The NSIS installer now bundles `vc_redist.x64.exe`. After Hope Agent installs, a POSTINSTALL hook silently runs Microsoft's official vcredist installer (a fast no-op on already-up-to-date systems). On top of that, Rust code is now built with `+crt-static` so it no longer depends on `VCRUNTIME140.dll` / `MSVCP140.dll` either — belt-and-suspenders, so hope-agent.exe runs even on a machine that has never seen a VC++ runtime. The download is verified against Microsoft's Authenticode signature in CI; if the POSTINSTALL hook fails it now pops a dialog with the `aka.ms` manual-download link instead of silently swallowing the error. **Existing v0.1.0 users** can still follow the README to install VC++ 2015-2022 Redistributable manually as a fallback.

- **`local_llm` watchdog spamming warnings on machines without Ollama** — The watchdog used to call `start_ollama` every 60 s regardless of whether the user had ever configured a local Ollama model, flooding logs with `Skipping pass — Ollama is unreachable`. It now bails out silently when no Ollama target exists, and warns exactly once on the transition into "unreachable" (resetting on recovery).

- **OpenClaw import compatibility** — Accept legacy string values for the `model` field in OpenClaw agent configs; OpenClaw `MEMORY.md` and SQLite memory chunks now import correctly into Hope core's `memory.md` and memory DB.

- **Settings-panel buttons squashed in narrow containers** — Six `Input + Button` rows (approval pattern lists / SSRF trusted hosts / Browser profile / Telegram group / per-channel "Test Connection") now have `Input` taking remaining space and `Button` keeping its natural width.

## Internal Naming Change (auto-migrated)

- **Default Agent ID renamed from `"default"` to `"ha-main"`** — On the first launch after upgrading to 0.1.1, **a one-shot migration runs automatically**: on-disk directories (`agents/default/` / `default-home/` / `plans/default/`), SQLite `agent_id` columns, `default_agent_id` / `recap.analysisAgent` / channel-level `agent_id` in `config.json`, and `agent_id` fields embedded in cron-job payloads all get renamed to `"ha-main"`. Each step is independently idempotent and crash-recoverable; a sentinel file ensures it doesn't re-run on subsequent launches. In the rare case a user has already hand-created `agents/ha-main/`, the migration aborts entirely and retries next launch — just glance at your sidebar agent name after upgrading to confirm.

## Documentation

- README (zh & en) gained a "macOS 'app is damaged / unidentified developer'" section and a "Windows missing-DLL on launch" section, plus a stronger framing of the cross-device / cross-endpoint motivation in the opening paragraph.

## Upgrading

- **Desktop**: install over the existing app; the agent-ID migration runs automatically on first launch
- **Server / Web mode**: replace the binary and restart; existing `hope-agent server install` units (systemd / launchd / Task Scheduler) don't need re-registration
- **Config / data**: compatible, no breaking changes

Same as v0.1.0 — see the [v0.1.0 release notes](./v0.1.0.en.md#known-limitations).
