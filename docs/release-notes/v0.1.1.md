# Hope Agent 0.1.1 — 维护版本

> **简体中文** · [English](./v0.1.1.en.md)

> 发布日期：2026-05-09

> 详见 [`CHANGELOG.md`](../../CHANGELOG.md#011---2026-05-09)。

本次为维护版本，核心是修复 v0.1.0 在纯净 Windows 上启动报 `MSVCP140_1.dll` / `VCRUNTIME140.dll` 缺失的问题，并附带几个独立修复与一项内部约定变更。无新功能，跨设备 / 跨入口的核心体验与 v0.1.0 一致。

## 核心修复

- **Windows 启动 DLL 缺失** — NSIS 安装包现在内嵌 `vc_redist.x64.exe`，主程序装完后由 POSTINSTALL hook 静默调用 Microsoft 官方 vcredist 安装器（已最新系统秒退）；同时给 Rust 自身代码加 `+crt-static` 静态链接 C runtime，与 vcredist 嵌入形成双保险，即使用户机器从未装过 VC++ 运行库也能跑 hope-agent.exe。下载流程 CI 端校验 `aka.ms` 文件的 Authenticode 签名，防中间人篡改；POSTINSTALL hook 失败会弹对话框并附 `aka.ms` 手动下载链接，不再静默吞错。**v0.1.0 老安装包用户**仍可按 README 提示手动安装 VC++ 2015-2022 Redistributable 兜底。

- **`local_llm` watchdog 在没装 Ollama 的机器上刷屏警告** — 之前 watchdog 每 60s 跑一次 `start_ollama`，没装 Ollama 也会被刷「Skipping pass — Ollama is unreachable」。现在入口先检查是否配过本地 Ollama 模型，没有就静默退出；真有目标但不可达时只在「转入不可达」那次警告一次，恢复后复位。

- **OpenClaw 导入兼容性** — 接受 OpenClaw agent 配置中 `model` 字段的 legacy 字符串值；OpenClaw `MEMORY.md` 与 SQLite memory chunks 现在能正确导入到 Hope core 的 memory.md 与 memory DB。

- **设置面板若干窄容器下按钮被挤压变形** — 审批面板模式列表 / SSRF 信任 host / 浏览器 profile / Telegram 群组 / 各 channel `Test Connection` 等 6 处 `Input + Button` 横排，统一为 `Input` 占满 + `Button` 保留自然宽度。

## 内部约定变更（自动迁移）

- **默认 Agent ID 从 `"default"` 改名为 `"ha-main"`** — 升级到 0.1.1 后**首次启动会一次性自动迁移**：磁盘目录（`agents/default/` / `default-home/` / `plans/default/`）、SQLite agent_id 列、`config.json` 里的 `default_agent_id` / `recap.analysisAgent` / channel 各级 agent_id、cron 任务的 payload，全部 rename 到 `"ha-main"`。每步独立 idempotent，崩溃可恢复；落 sentinel 文件防止重复执行，不影响后续启动速度。少数情况下若用户已手动建过 `agents/ha-main/`，本次迁移会整体放弃，下次启动重试——升级后看一眼侧边栏 Agent 名是否正常即可。

## 文档

- README 中英双语补「macOS 启动报『已损坏 / 无法验证开发者』」与「Windows 启动报 DLL 缺失」两段处理说明，缘起段强化「跨设备、跨入口接续」动机。

## 升级

- **桌面**：直接覆盖安装；首次启动自动跑 Agent ID 迁移
- **Server / Web 模式**：替换二进制后重启服务，`hope-agent server install` 注册过的 systemd / launchd / 任务计划无需重做
- **配置 / 数据**：兼容，无破坏性变更
