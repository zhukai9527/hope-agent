# Hope Agent 0.1.0 — First Public Release

> [简体中文](./v0.1.0.md) · **English**

> Release date: 2026-05-08
> A local AI assistant desktop app built on Tauri 2 + React 19 + Rust — and also runs headless as a long-lived daemon on your server.

> Note: 0.1.0 content was prepared by subsystem starting 2026-04-19; the actual first release happened 2026-05-08. Fixes and polish accumulated in between (permission / approval v2 redesign, Plan Mode decoupling, local-model auto-maintenance watchdog, removal of message-feed virtualization, CLI setup wizard parity with the 11-step GUI, IM ↔ GUI live bidirectional streaming mirror, IM reply split/preview/final modes, `/thinking` as the primary slash command, cross-device session handover, desktop markdown path links, `@` file mentions, IM channel auto-start watchdog, file Diff panel, projects as first-class sidebar items, Skill self-evolution tab, dozens more) are all rolled into this release. See [`CHANGELOG.md`](../../CHANGELOG.md#010---2026-05-08) for the full list.

One native installer. Fill in an API key and start chatting. It remembers across sessions, tidies up its own memory while idle, distills completed tasks into reusable skills, and plugs into the IM apps you already live in.

## Highlights

- **Three run modes, one core**: desktop GUI (default) / `hope-agent server` (HTTP/WS daemon, installable as a system service with autostart) / `hope-agent acp` (stdio, as an IDE agent backend)
- **Remote-reachable**: in server mode, phones, tablets, browsers, and other desktops can all connect to the same backend — Bearer Token + three-tier SSRF policy
- **36 Provider templates · 165 preset models**: Anthropic / OpenAI / Codex (ChatGPT OAuth) / Gemini / OpenRouter / DeepSeek / Kimi / Qwen / Doubao / GLM / Ollama — everything wired up out of the box
- **Multi API-key auto-rotation**: multiple keys per Provider seamlessly switch on rate limit / quota exhaustion / auth failure; only if all are exhausted does it fall through to the next model
- **12 IM channels**: Telegram / Discord / Slack / Feishu / Google Chat / LINE / QQ Bot / Signal / iMessage / IRC / WeChat / WhatsApp. Inbound images / voice / files land in the multimodal context automatically; tool approvals can be clicked right inside the chat window
- **Cross-session persistent memory**: SQLite + FTS5 full-text + vector search combined, three-tier scope (global / project / agent), joint budget allocation so tiers don't squeeze each other out
- **Active recall + reflective profile + offline "dreaming"**: each turn pulls relevant memories into the prompt based on what you just said; reflection extracts a user profile from past conversations; on idle it auto-curates the most valuable memories from the past N days into a markdown diary
- **Skills that grow**: after finishing a complex task, drafts a new skill for your review; supports `paths:` conditional activation, fork-into-subagent execution, and per-skill tool allowlists; compatible with the agentskills.io open standard
- **Project container**: group related sessions together with project-level memory / instructions / shared files; uploaded files are auto text-extracted and loaded on demand
- **Agent Team multi-agent collaboration**: preset team templates in Settings — one natural-language line and the model spins up a specialist team to work in parallel
- **Plan Mode + natural-language cron + cron results piped to IM**: say "write me a daily report at 8 a.m." and the result can land right back in Telegram / Feishu when it fires
- **Five-tier progressive context compaction**: no matter how long the conversation runs, it never hard-cuts; tool_use / tool_result pairs are never split across the boundary; combined with prompt caching, long-session API cost drops dramatically vs. naïve calls
- **Background long-running tasks**: slow shell / web search / AI image generation can be dispatched to the background, returning `job_id` immediately without blocking the chat; results auto-inject when done
- **Built-in toolbox**: remote-controlled browser (CDP) / Canvas / AI image generation (7 Providers) / web search (8-Provider failover) / bash (optional Docker sandbox) / file read-write / grep / MCP / URL preview
- **Dashboard + `/recap` deep review**: 4-axis visualization (cost / tokens / activity heatmap / health score); one-click 11-section AI report exportable as self-contained HTML

## Download & Install

Installers are attached to this Release — pick your platform and double-click. No need to pre-install Node, Rust, or Python.

- **macOS**: if the first launch complains about an unsigned app, allow it via **System Settings → Privacy & Security**
- **Windows**: installers are **not yet code-signed**, so SmartScreen may block them — choose **More info → Run anyway**. ConPTY is unavailable on Windows 10 1809 and earlier, which triggers a PTY fallback
- **Linux**: for AppImage, `chmod +x` and run directly; for deb, `sudo apt install ./hope-agent_*.deb`

## First Launch

1. In the onboarding wizard, pick a Provider template → fill in the API key (Codex supports ChatGPT OAuth login) → start chatting
2. The embedded HTTP server binds to `127.0.0.1:8420` by default (local-only). Change `server.bindAddress` in `~/.hope-agent/config.json` to `0.0.0.0:8420` to expose it on your LAN (strongly recommended to also set `server.apiKey`)
3. `hope-agent server install` registers the daemon as a system service with autostart — launchd on macOS, systemd user unit on Linux, Task Scheduler on Windows (user-level `/SC ONLOGON` task, not a true Windows Service)

## Security

- **Unified SSRF policy**: every outbound HTTP request that consumes model / user URL input goes through `security::ssrf`, with three tiers `Strict` / `Default` / `AllowPrivate` and a `trustedHosts` allowlist. Metadata IPs (cloud-provider metadata endpoints, link-local) are rejected in every tier
- **Tool approval gating**: three states `ask` / `auto` / `full-approve`, session-persistable; supports timeout-to-deny / proceed; native approval buttons inside IM channels; optional per-channel auto-approve
- **Dangerous Mode (YOLO)**: a process-level "nuke button" that skips all tool approvals; requires an explicit "I understand the risk" checkbox in Settings to enable; a red banner stays pinned at the top of the window while active
- **API-key auth**: `Authorization: Bearer` header + `?token=` query fallback for browser WebSockets that can't set custom headers; `api_key: null` on a localhost-only bind means "allow all"
- **CSP now enforced**: the Tauri WebView CSP moved from `null` to a full policy, with all inline scripts removed
- **Automatic config snapshots**: every config write goes through `config::save_config()`, which snapshots the old file into `~/.hope-agent/backups/autosave/` (last 50 kept) — roll back any change anytime

## Known Limitations

- **iMessage channel is macOS-only** (depends on system AppleScript and the iMessage database)
- **macOS native geolocation is macOS-only** (CoreLocation delegate); other platforms fall back to IP-based geolocation for weather
- **Windows "system service" is actually a Task Scheduler user task**, not a true Windows Service (we avoid pulling in SCM dispatch code — users who need a real service can wrap the binary with nssm)
- **Windows ConPTY requires Windows 10 1809+**; older versions fall back from the PTY terminal
- **Windows installers are unsigned**, so SmartScreen may block them
- **The bundled Docker SearXNG, when searching private-network addresses** (e.g. `192.168.x.x`), will be rejected by the `Default` SSRF policy — add the host to `ssrf.trustedHosts`. The default `127.0.0.1:8080` loopback is not affected
- **Cron-driven `/recap` reports + `recap --export` CLI subcommand**: not wired up in Phase 1 — only manual / Dashboard Tab / `/recap` slash command are available
- **Multi-window / native notifications / system tray / global shortcuts / autostart**: not available in HTTP/Web mode (Tauri-only); the server mode presents these as graceful degradation
