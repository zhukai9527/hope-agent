The Hooks system lets you wire custom automation into the key lifecycle events of a conversation; a new Workspace panel on the right aggregates a task's progress, the files it touched, and the sources it referenced; the project file subsystem has been rebuilt around a real working directory, a VSCode-style browser, and unified file actions with built-in preview; and macOS desktop control advances to v3 with visual / OCR targeting and a systematic reliability pass.

## Hooks System (field-aligned to the Claude Code hooks protocol)

Attach custom handlers to **28 lifecycle events** ‚Äî across conversations, tool calls, context compaction, permission decisions, and more ‚Äî to weave external automation into the chat:

- **Five handler types**: `command` (run a command) / `http` (SSRF-checked HTTP callback) / `mcp_tool` (invoke an MCP tool) / `prompt` (side-query) / `agent` (spawn a sub-agent)
- **Four scopes that stack**: user / managed / project / local. Project-scope hooks can ship with the repo for the whole team; for supply-chain safety they are **off by default** and must be enabled explicitly in Settings ‚Üí Hooks
- **Visual settings panel**: configure under Settings ‚Üí Hooks, with hot reload on change
- **Fine-grained control**: conditional execution (`if`, matched by tool + argument pattern), run-once-per-session (`once`), a runtime desktop hint (`statusMessage`), and `asyncRewake` for async command hooks to feed their result back as a reminder on the next turn when they exit with code 2

## Workspace Panel & File Subsystem

- **Workspace panel**: the right side of the chat now aggregates the session's task progress, the files it touched (reads + edits ‚Äî click to view the diff or locate them in the file browser), and referenced sources (web-search hits and in-body links). Files and sources cover the **entire session history** while still reflecting the in-flight turn in real time; task progress moves from above the input box into the panel, leaving only a "Running N/M" status bar. It appears automatically the first time it has content, and won't re-open on its own once dismissed
- **Project file browser**: both the project Settings "Files" tab and the main chat area offer a VSCode-style browser (titlebar entry + right panel, openable as a standalone window or maximized), with a resizable split between tree and preview; it supports create / rename / delete / move / drag-and-drop upload, plus read-only preview of code (Shiki highlighting), Markdown, images, PDF, and Office (docx/xlsx/pptx). When the working directory is a git repo it shows the current branch and worktree list with read-only navigation; selected text in a preview can be copied or referenced into the conversation
- **Unified file actions + built-in preview**: Markdown links, message attachments, and workspace artifact files now behave identically ‚Äî previewable types open in the right-side preview panel, everything else opens locally or downloads remotely. Local files inside the working directory or referenced by the session are no longer disabled in HTTP / Web mode
- **Colored file-type icons**: the workspace, message attachments, and file browser all switch to per-format colored icons for at-a-glance recognition

> ‚öÝÔ∏è **Breaking change**: every project now has a real working directory (default `~/.hope-agent/projects/<id>/workspace/`), and uploaded files land directly in it. The old project-file upload mechanism (separate storage + automatic text extraction) has been removed; after upgrading, previously uploaded project files are no longer injected into the model or shown in the UI.

## macOS Desktop Control v3

Building on v2's "can click and type", v3 adds visual awareness and hardens execution reliability:

- **Visual / OCR targeting v1**: `visual.observe/point` returns a managed screenshot as visual input and maps pixels to clickable screen points; `visual.ocr/find_text` recognizes on-screen text via the Vision Framework and returns clickable coordinates; `observe` supports `annotate=true` to return an annotated screenshot with AX element-id boxes
- **Higher-level actions**: `act.perform_action` runs named AX actions (`AXPress` / `AXShowMenu` / `AXIncrement`, etc.); `dialog` gains `list/click/input/file`; new `dock` and `spaces` families manage Dock items and desktop Spaces
- **Reliability hardening**: `visual.point/find_text` return AX-first `suggestedActions[]`; `act` supports stable re-targeting via `snapshotId + elementId`, observe‚Üíact‚Üíverify checks, and post-approval focus restoration; arguments are sanitized + preflighted before approval; Spaces switching and current-state detection move to more reliable implementations

## Experience Polish & Fixes

- **Approval / ask-user timeouts (opt-in)**: optional timeouts for tool approval and `ask_user_question`; on timeout the session auto-denies or proceeds per your config, so it won't get stuck when unattended
- **Sidebar display modes**: a new compact / detailed toggle, with compact mode tightening list density
- **User message attachments** are now shown alongside the model's; a new **update history page** lets you revisit each version's release notes
- **Right-panel interaction & animation polish**: panels gain a collapse toggle and a tab-style switcher, and the left session sidebar stays open when a panel is open
- **Streaming Markdown render performance**: fixes dropped frames late in long replies; fixes project-wide tooltips being clipped by an ancestor's `overflow`
- **A batch of fixes**: subagent / parent-agent streaming, AllowAlways permission scopes, pinned-session migration compatibility, Web Search fallback diagnostics no longer polluting the cache, and more
