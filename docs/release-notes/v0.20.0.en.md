# Hope Agent 0.20.0 — Design Space and an End-to-End Delivery Loop

> [简体中文](https://github.com/shiwenwen/hope-agent/blob/v0.20.0/docs/release-notes/v0.20.0.md) · **English**

> Released: 2026-07-19

> See [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.20.0/CHANGELOG.md#0200---2026-07-19) for the full list.

v0.20.0 jumps straight from v0.17.0, folding in what was planned as 0.18 and 0.19 — 58 commits in total. The headline is the new **Design Space**: an agent-native design workspace that turns a one-sentence brief directly into deliverable artifacts — web pages, mobile prototypes, decks, posters, interactive components, audio — and can link to a real code repository so a design lands as component code in one step. Around that headline, this release adds a local-first **Artifacts platform** with structured data-analysis reports, introduces an **agent control plane** built from `/goal`, `/mode`, `/workflow`, and `/loop`, consolidates image and audio generation into a single "provider → models → per-capability default chain" system, upgrades memory to **Memory UX v2**, and completes a full Git and PR control surface in the workspace. File, attachment, and upload handling is now unified as well, behaving identically on local desktop, remote desktop, and the web.

## Highlights

- **The new Design Space subsystem.** A dedicated entry below "Knowledge Space" in the sidebar: type one sentence and watch a self-contained artifact take shape as it streams — web page, mobile prototype, deck, dashboard, poster, document, email, image, motion, audio, or a real React interactive component. When the brief is vague the AI asks about audience and purpose; when a visual direction is needed it presents style cards with real palette and type samples. Six original design systems and a set of mainstream brand-style references can be swapped in with one click, and a design system can be extracted in reverse from a screenshot, a URL, an existing codebase, or a Figma file. Artifacts support click-to-tweak visual editing, double-click inline text editing, annotation pins and region markup, and export to HTML / PNG / PDF / PPTX / MP4 / ZIP or deployment to Cloudflare Pages.
- **Designs and code repositories, connected in both directions.** A design project can be linked to a real repository, and the artifact menu gains "Implement to code…": the design, together with your implementation requirements, is handed to the main conversation and lands as component code in your existing stack, with every file change going through normal approval and the diff panel. The reverse holds too — if you later change those files in the repository, Design Space notices and flags the corresponding artifact, so you can review the diff, have the AI update the design from the latest code, or mark it as synced.
- **A new agent control plane: goals, modes, dynamic workflows, and loops.** Four new slash commands and their supporting tools: `/goal` (declare the end goal and its required completion criteria, then audit progress against evidence), `/mode` (set a persistent execution intensity), `/workflow` (orchestrate a task as a previewable, approvable, observable execution script that survives interruption), and `/loop` (repeat on an interval or event, with a no-progress guard). A unified control panel in the right-hand workspace shows goal progress and evidence, workflow timelines and approval records, loop run history, and code review and verification results.
- **Image and audio generation unified into a provider system.** Generation is now configured as "provider → models → per-capability default chain", with providers and credentials managed centrally in model settings. Built-in templates grew from 9 to 27, covering OpenAI, Google, Volcano Ark, Tencent Hunyuan, Fal, MiniMax, ElevenLabs, Black Forest Labs and more, plus custom OpenAI-compatible endpoints. A new `audio_generate` chat tool produces speech, music, and sound effects directly in conversation.
- **A local-first Artifacts platform and native data analysis.** On top of the existing Canvas runtime, artifacts gain a unified identity, immutable versions, content addressing, version restore, and provenance summaries — entirely on your machine, with no public deployment required; existing Canvas projects remain compatible as-is. Alongside it, native data analysis handles CSV / XLSX, project files, and Knowledge Space material, producing structured reports with metric definitions, findings, charts, data-quality checks, and source auditing. Key numbers must be recomputable, and insufficient data is stated plainly rather than invented.
- **Memory upgraded to Memory UX v2.** Global, agent, and project core memory all enter the prompt as a stable snapshot, while detailed topics stay in dynamic storage and are read on demand. New additions include core-memory management, review of pending memories, and four learning modes. Automatic dynamic recall is now explicitly opt-in — by default only bounded core memory is used; once enabled, a deterministic fast recall runs with no extra model cost, and the slower deep recall stays off by default.
- **A complete Git and PR control surface in the workspace.** Git status, run location, and managed worktree lifecycle are consolidated into a single control card: review staged and unstaged diffs, stage / unstage / discard by all, file, or hunk, switch branches, and commit, push, and open a draft PR. Linked PRs get their own right-hand panel showing review comments, check details, and unresolved inline comments, and any or all feedback can be pulled into the current session to fix.

## Upgrade notes

Two settings are deliberately not migrated in this release and must be reconfigured after upgrading:

- **Image / audio generation settings are not migrated.** The old "Settings → Tools → Image generation / Audio generation" configuration is not converted to the new provider system. After upgrading, add your providers and credentials again under "Settings → Models → Media generation models", then confirm the four per-capability default chains under "Settings → Tools → Media generation".
- **Project instructions are not migrated.** Project instructions have moved out of the database and into an `AGENTS.md` file in the project root. Existing content is not carried over and must be re-entered in `AGENTS.md`.

## Files, attachments, and artifacts

- **Unified file and attachment handling across local desktop, remote desktop, and web.** The file tree, message attachments, input drafts, tool media, workspace artifacts, and Artifacts all share the same file-type, preview, open, download, edit, and context-menu behavior. Local desktop acts on your own machine; remote desktop and web act on the server filesystem. Unsent attachments always stay on the client and can be previewed or edited offline.
- **Configurable file size limits and unified chunked upload leases.** Settings now expose limits for chat attachments, workspace uploads, text preview / editing, document preview, Artifact import, and knowledge sources. Client-side files move to 4 MiB chunked, resumable temporary leases with concurrency caps, failure reclamation, and completion checks — web and remote desktop no longer buffer whole files at once.
- **Consistent artifact library navigation, layout, and status labels.** The sidebar entry moved below the dashboard and is now consistently named "Artifacts"; the artifact list and property panel can be collapsed and resized with preferences remembered. Type, privacy, and analysis / verification status no longer leak English snake_case. Local export no longer asks for an "intended recipient" — only sensitive sources or executable content prompt a one-time risk confirmation at export.

## Conversation and collaboration

- **Inline subagent capsules and a dedicated Subagents panel.** Subagents in a conversation are now compact capsules instead of full cards; opening one reveals a right-hand Subagents panel that groups every subagent in the session by running / completed, shows full invocation details, and supports live conversation with the subagent session and drilling into nested subagents.
- **Quote selected text into the input box.** Select text in a user message or assistant reply and right-click to copy the selection or add it to the conversation. Quotes are held as removable cards, can be added consecutively, and survive editing.
- **Chat message queueing is now durable and recoverable.** Messages sent while a reply is generating are written to the local database along with their attachments, with real queue state shown above the input box. The queue survives app restart, page reload, and concurrent clients without sending anything twice.
- **Floating windows and execution history for the browser and macOS control panels.** Both live mirror panels can become floating in-app windows; docked, they gain an execution-history timeline, session statistics, and a quick-action bar.
- **Unread counts unified to "unread conversations".** A conversation counts once no matter how many replies are pending. A conversation counts as read only when the window is in the foreground, the UI is visible, and the message list is at the latest position — replies arriving while you're elsewhere, unfocused, or scrolled back are no longer silently marked read.

## Models and providers

- **Refreshed built-in provider templates.** Adds current flagships including Claude Sonnet 5 and Mythos 5, GPT-5.6, Kimi K3, Grok 4.5, and Qwen 3.7 Plus, plus five new providers (Cohere, Baseten, LongCat, Meta, Featherless), bringing built-in templates to 50 providers and 392 preset models.
- **Providers can declare a pricing currency.** A new USD / CNY setting lets you enter prices exactly as published; the cost dashboard converts to USD for reporting, so CNY price lists are no longer booked as dollars.
- **Model selection is now per-session.** Changing model, reasoning effort, or temperature in the input-box picker affects only the current session; you can explicitly "also set as agent default" when you want it to stick. If a preferred model's provider is disabled or removed, your choice is preserved and flagged at the top of the picker instead of being silently rewritten.
- **One-off background AI tasks get their own model configuration and fallback.** Recap, Dreaming, Knowledge Space maintenance, skill auto-review, image OCR, sprite mode, note tools, and recall summaries can share one default model chain or override it individually, and now automatically try the next model in the chain when one is unavailable.
- **Context accounting and deferred tool loading follow a provider-aware contract.** `/context`, compaction decisions, cache reads and writes, and the dashboard all use the same actual-request accounting. The tool catalog splits into eager / deferred / activated, and `tool_search` now genuinely activates a tool so it is callable on the next turn.

## Dashboard and settings

- **A new unified "Goals and execution" page.** Goal attainment and required criteria, workflow completion / failure / approvals, loop progress and no-progress guards, task and plan progress, and items currently needing attention, all in one place — filterable by time, agent, and project, with risk items linking back to the conversation.
- **Costs are now settled against the prices you configured.** The dashboard previously ignored the per-model input / output prices entered in the provider editor and always used the built-in estimate table. It now prefers your configured price keyed on provider + model, so the same model reached through different routes is billed separately. Prices also distinguish "unpriced" from "explicitly free": local models and subscription endpoints are recorded as $0 rather than charged at an estimated rate.
- **Usage can be broken down by purpose.** Token usage trends gain two levels of breakdown — by capability group and by precise purpose tag — with drill-down to the corresponding token and cost detail.
- **Settings support layered restore to defaults.** Each settings page, tab, and individual section has its own "restore defaults" entry, with a confirmation that explains the scope. Providers, credentials, and existing memory, knowledge spaces, tasks, and artifacts are never deleted.
- **A shorter first-run setup.** Initial configuration is down from 11 steps to 6, and connecting to a remote server has moved to an expandable secondary entry on the welcome screen. Users mid-onboarding when they upgrade resume at the nearest remaining step.

## Interface and interaction

- **Unified focus feedback and form control styling.** Mouse and touch interaction no longer produce an abrupt dark focus ring, keyboard navigation keeps a light indicator, and an enhanced focus mode can be turned on manually. Search boxes, inputs, selectors, and number fields share one flatter, more restrained surface.
- **The right-hand panel entry is now a single dock.** The previously separate "Files", "Workspace", and "Background jobs" buttons and the panel switcher merge into one dock where every available panel appears as an icon in the same row.
- **Consistent overlay behavior in the input area.** The "+" menu and the permission and sandbox settings all adopt the model picker's overlay styling and animation, with sandbox settings folded into the permission overlay and collapsed by default.
- **Consistent list row feedback.** Hover and selected states in Knowledge Space and scheduled tasks use neutral tones rather than a light blue background or blue text; status, error, unread, and drag-target colors keep their own semantics.
- **Panel resizing feels tighter.** Drag handles in the chat sidebar, right-hand panel, file browser, and Knowledge Space have larger hit areas and highlight the panel border on hover and drag, and panels now track the cursor exactly without animation lag.

## Fixes and stability

- **Memory reads and writes no longer deadlock or stall the conversation.** Fixes a hard hang when saving, updating, or searching memory caused by the embedding cache and database connections waiting on each other. Automatic memory and knowledge recall now runs on background threads with timeouts and a concurrency cap, so a slow disk or a large memory store at worst injects a few fewer references this turn.
- **Approval and question countdowns, stop, and dismissal are now accurate.** Approval dialogs and question cards use a backend-authoritative deadline, so the countdown stays correct across page reloads and app restarts. Pressing "stop" or deleting a session immediately rejects any in-flight approval or question and dismisses it on desktop, web, and IM at once.
- **Loop task status and labeling in scheduled tasks corrected.** Loops created by `/loop` no longer inherit the underlying trigger's status — a finished loop now shows as "completed" rather than being misreported as a resumable "paused" task (historical data is corrected automatically on first launch after upgrading), and task names no longer leak the internal `[Loop] ` prefix.
- **Corrected pricing and context length for several models.** Claude Opus 4.8 / 4.7 / 4.6 were billed at the older $15/$75 tier (actually $5/$25), and Opus 4.7 / 4.6 and Sonnet 4.6 still listed a 200K context (actually 1M) — an inflated context delays compaction until the request hits the real limit and errors. Context and pricing for DeepSeek, Mistral Small, and the GPT-5.6 series are corrected as well.
- **Interface localization completed.** Remaining hardcoded English strings and raw English status values across the dashboard, Knowledge Space, workspace, teams, and the settings panels are now localized in all 12 languages, as are the names and descriptions of roughly 60 newer tools shown in settings and the agent capabilities page.
- **More reliable clicking on macOS.** The main window, quick chat, Plan, file browser, and Canvas panels respond to the first click even when the window is inactive, and a WebKit quirk that occasionally reversed `pointerup` / `pointerdown` under some input methods no longer forces a second click.
- **Safer Git handoff and commit failure recovery.** Worktree targets are strictly limited to those owned by the current session or its subsessions, and failed rollbacks only revert changes captured in the snapshot, leaving unrelated files intact. When a commit succeeds but an optional push fails, the commit is kept and a clear warning is returned instead of reporting the whole operation as failed.
- **Fixed Cask installation being rejected on older Homebrew.** The cask's macOS requirement now uses the "at least Big Sur" form; older Homebrew read the previous form as "Big Sur only" and failed the install on newer macOS.
- **Unified two-way transitions for in-app fullscreen panels.** Canvas, the file browser, single-file preview, Plan, and the artifact reader all expand smoothly from their current layout position and collapse back on restore, switching instantly when "reduce motion" is enabled.
