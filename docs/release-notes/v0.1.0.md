# Hope Agent 0.1.0 — 首次公开发布

> **简体中文** · [English](./v0.1.0.en.md)

> 发布日期：2026-05-08
> 基于 Tauri 2 + React 19 + Rust 的本地 AI 助手桌面应用，也能脱离 GUI 作为守护进程常驻服务器。

> 注：0.1.0 内容于 2026-04-19 起按子系统准备完毕，2026-05-08 实际首发；其间累积的修复与打磨（权限/审批 v2 重构、Plan Mode 解耦、本地模型自动维护 watchdog、消息流虚拟化撤回、CLI setup 引导补齐 GUI 11 步、IM ↔ GUI 实时双向流式镜像、IM 回复 split/preview/final 三态、`/thinking` 主命令、会话跨端交接、桌面 markdown 路径链接、`@` 文件引用、IM channel 自启动 watchdog、文件 Diff 面板、项目侧边栏一等公民、Skill 自我进化 Tab 等数十项）一并合并发布，详见 [`CHANGELOG.md`](../../CHANGELOG.md#010---2026-05-08)。

一个原生安装包，填完 API Key 就能开聊。会跨会话记忆、能空闲时自己整理记忆、能把做过的任务沉淀成可复用的技能，还能接进你常用的 IM。

## 亮点

- **三种运行模式同核**：桌面 GUI（默认）/ `hope-agent server`（HTTP/WS 守护进程，可注册为系统服务开机自启）/ `hope-agent acp`（stdio，给 IDE 当 agent 后端）
- **远程可连**：服务模式下手机、平板、浏览器、另一台电脑都能连回同一个后端，Bearer Token + SSRF 三档策略
- **36 个 Provider 模板 · 165 个预设模型**：Anthropic / OpenAI / Codex (ChatGPT OAuth) / Gemini / OpenRouter / DeepSeek / Kimi / Qwen / 豆包 / GLM / Ollama …一站配齐
- **多 API Key 自动轮换**：同一 Provider 下多把钥匙，遇到限流 / 额度耗尽 / 认证失效无缝切换，不够再跳下一个模型
- **12 个 IM 渠道**：Telegram / Discord / Slack / 飞书 / Google Chat / LINE / QQ Bot / Signal / iMessage / IRC / WeChat / WhatsApp。图片 / 语音 / 文件入站自动进多模态上下文，工具审批在聊天窗直接按按钮决定
- **跨会话持久记忆**：SQLite + FTS5 全文 + 向量三位一体，全局 / 项目 / Agent 三层 scope，联合预算分配不互相挤占
- **主动召回 + 反省画像 + 离线"做梦"**：每轮对话按你刚说的话主动捞相关记忆注入；反省历史对话提炼用户画像；空闲时自动整理过去 N 天最有价值的记忆并写成 markdown 日记
- **会成长的技能系统**：执行完复杂任务自动生成技能草稿等你审核；支持 `paths:` 条件激活、fork 子 Agent 执行、工具白名单隔离；兼容 agentskills.io 开放标准
- **Project 项目容器**：把相关会话归到一起，继承项目级记忆 / 指令 / 共享文件；上传的文件自动文本抽取并按需读取
- **Agent Team 多 Agent 协作**：设置里预置团队模板，模型一句话就能组建专家团协同推进
- **Plan Mode + 自然语言定时任务 + Cron 结果推送 IM**："每天早 8 点给我写日报" 到点自动跑，结果可直接发回 Telegram / 飞书
- **五层渐进式上下文压缩**：不管聊多久都不会被强切丢失，tool 调用配对永远不拆散；配合 Prompt Caching 长会话 API 成本显著低于朴素调用
- **后台长任务**：耗时 shell / Web 搜索 / AI 画图可"丢到后台跑"，立即返回 `job_id` 不阻塞对话，完成后自动注入结果
- **内置工具箱**：可控浏览器（CDP）/ Canvas / AI 画图（7 Provider）/ Web 搜索（8 Provider failover）/ bash（可选 Docker 沙箱）/ 文件读写 / grep / MCP / URL 预览
- **Dashboard + `/recap` 深度复盘**：成本 / Token / 活跃热力图 / 健康度四维可视化；一键生成 11 章节 AI 报告导出独立 HTML

## 下载与安装

安装包挂在本次 Release 附件里，对应平台双击运行即可——不需要预装 Node、Rust 或 Python。

- macOS：首次打开若提示未签名，请在"系统设置 → 隐私与安全性"放行
- Windows：安装包**目前尚未做代码签名**，SmartScreen 可能拦截，选择"更多信息 → 仍要运行"；Windows 10 1809 以下系统的 ConPTY 不可用会降级
- Linux：AppImage 赋执行权限后直接运行；deb 包用 `sudo apt install ./hope-agent_*.deb`

## 首次启动

1. 启动后向导里选 Provider 模板 → 填 API Key（Codex 可走 ChatGPT OAuth 登录）→ 开聊
2. 桌面应用内嵌的 HTTP 服务默认绑定 `127.0.0.1:8420` 仅本机访问；改 `~/.hope-agent/config.json` 的 `server.bindAddress` 为 `0.0.0.0:8420` 可对外暴露（建议同时设置 `server.apiKey`）
3. `hope-agent server install` 可把守护进程注册为系统服务开机自启——macOS 走 launchd、Linux 走 systemd user unit、Windows 走 Task Scheduler（用户级 `/SC ONLOGON` 任务，非真正 Windows Service）

## 安全

- **SSRF 统一策略**：所有面向模型 / 用户 URL 输入的出站 HTTP 走 `security::ssrf`，三档 `Strict` / `Default` / `AllowPrivate` + `trustedHosts` 白名单；元数据 IP（云厂商 metadata、link-local）所有档位都拒
- **工具审批门控**：三态 `ask` / `auto` / `full-approve`，可会话级持久化；支持超时自动 deny / proceed；IM 渠道原生按钮审批；渠道级自动批准可选
- **Dangerous Mode (YOLO)**：一键跳过所有工具审批，进程级"核弹按钮"，需要在设置里二次确认勾选"我已了解风险"才能启用；开启后标题栏常驻红条提示
- **API Key 鉴权**：`Authorization: Bearer` header + 浏览器 WebSocket 用 `?token=` query 回退，默认 `api_key: null` 本机绑定时全放行
- **CSP 首次落地**：Tauri WebView CSP 从 `null` 收紧到完整策略，移除所有 inline 脚本
- **配置自动快照**：任何配置写入都通过 `config::save_config()` 自动在 `~/.hope-agent/backups/autosave/` 存档，保留最近 50 份，可随时回滚

## 已知限制

- **iMessage 渠道仅 macOS**（依赖系统 AppleScript 和 iMessage 数据库）
- **macOS 原生定位仅 macOS**（CoreLocation delegate）；其他平台天气走 IP 地理位置降级
- **Windows "系统服务"实际是 Task Scheduler 用户任务**，不是真正 Windows Service（避免引入 SCM dispatch 代码，需要真正系统服务的用户可手动用 nssm）
- **Windows ConPTY 需要 Windows 10 1809+**；老版本下 PTY 终端降级
- **Windows 安装包未代码签名**，SmartScreen 可能拦截
- **内置 Docker SearXNG 若向私网地址搜索**（如 `192.168.x.x`），会被 `Default` SSRF 策略拒绝，需把 host 加入 `ssrf.trustedHosts`；默认 `127.0.0.1:8080` loopback 不受影响
- **cron 自动生成 `/recap` 报告 + CLI `recap --export` 子命令**：Phase 1 未接入，仅手动 / Dashboard Tab / `/recap` 斜杠命令可用
- **多窗口 / 原生通知 / 系统托盘 / 全局快捷键 / autostart**：HTTP/Web 模式下不可用（Tauri-only），服务器模式以 graceful degradation 呈现
