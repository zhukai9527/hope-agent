# Hope Agent 0.21.0 — 内置手册、能力评测与流式持久化

> **简体中文** · [English](https://github.com/shiwenwen/hope-agent/blob/v0.21.0/docs/release-notes/v0.21.0.en.md)

> 发布日期：2026-07-20

> 详见 [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.21.0/CHANGELOG.md#0210---2026-07-20)。

v0.21.0 是一个巩固版本，主线是把「用起来」和「信得过」这两件事补齐：完整的中英双语用户手册随应用内置，随时可查、也可以直接问 AI；数据大盘新增能力评测中心，可以用你自己的模型跑一遍 Hope Core 场景，看清它在真实任务上的表现；流式回复改为先落盘再广播，崩溃、重载或中途停止都不再丢内容。

## 亮点

- **内置用户手册与帮助中心**：完整的中英双语使用手册随应用内置，无需联网即可查阅。侧边栏、关于页、macOS 帮助菜单与托盘都能一键打开独立的帮助窗口，支持全文搜索、章节大纲与章节内查找；设置页每个分区的「?」可直达对应章节，选中手册里的任意内容还能一键「问 AI」带着上下文提问。AI 助手本身也能查阅这份手册，直接回答「这个功能怎么用」一类的问题。
- **数据大盘新增「能力评测」中心**：可以用已配置的服务商、本地模型或已登录的 Codex 运行 Hope Core 能力评测，覆盖目标推进、工作流、并发异步任务与子 Agent 协作等核心场景。运行前可预览计划并设定费用与时长预算，运行中可查看逐场景的实时进度并随时取消，运行后可翻阅逐场景详情、对比不同版本、查看趋势与添加备注，结果也可导入导出。失败的场景会说明实际消耗、解析后的预算上限、触发维度与因果轨迹，而不只给一个失败标记。取消或中断的实验不支持原地恢复——已发往服务商的请求与计费无法安全冻结——但可以从历史「重试为新实验」按原请求重跑，父记录与已产生的费用证据都会保留。发起评测是桌面端操作，Server / Web 模式只提供脱敏的只读查询；本机结果仅作诊断用途，不会成为发布门禁证据。
- **回复内容不再因崩溃、重载或中途停止而丢失**：流式回复改为先落盘再广播——助手消息、上下文与用量在同一个事务里落定，应用崩溃或被强制退出后重启会依据落盘记录恢复到一致状态。页面重载不再丢失、重复或回滚已经收到的内容，服务商全部失败、点击停止与自动降级等收尾路径也走同一套恢复逻辑。无痕会话仍然全程零落盘。
- **清理了一个已停用的内嵌推理引擎**：移除旧版本地 embedding 实现残留的 ONNX 运行时，内嵌前端资源改用 brotli 压缩，合计为安装包减重约 10MB。本地 embedding 功能不受影响，仍由「设置 → 记忆 → 本地 embedding 助手」的 Ollama 方案提供。需要说明的是，本版新增的内置手册与能力评测资产体积更大，抵消并超过了这部分收益——各平台安装包相比 v0.20.1 仍净增约 7–10MB。

## 修复与稳定性

- **内置技能与浏览器扩展在所有部署形态下不再缺失或过时**：内置技能与浏览器扩展改为编译进主程序，桌面、Docker 与单文件三种形态统一携带并随升级自动更新；单文件发布包一并附带浏览器扩展桥 `ha-browser-host`，自升级时同步换新。升级后，Chrome 中已加载的扩展需要在 `chrome://extensions` 手动 reload 一次才会生效。
- **设置页默认打开首个分区**：从侧边栏齿轮进入设置时停在「个人设置」，不再固定跳到「模型设置」；从其它入口显式跳转到指定分区的行为不变。
- **侧边栏会话加载动画不再冻结**：运行中的会话在侧栏首次载入后会持续显示转圈与待处理脉冲，不必先与侧栏交互一次才恢复动态。
