# Hope Agent 0.16.0 — 知识空间「来源 → 笔记」

> **简体中文** · [English](https://github.com/shiwenwen/hope-agent/blob/v0.16.0/docs/release-notes/v0.16.0.en.md)

> 发布日期：2026-07-06

> 详见 [`CHANGELOG.md`](https://github.com/shiwenwen/hope-agent/blob/v0.16.0/CHANGELOG.md#0160---2026-07-06)。

v0.16.0 的主线是为**知识空间**补上一条完整的「来源 → 笔记」编译工作流：把网页、聊天媒体、本地文档等学习材料先归档为「来源」，再由 Agent 整理成可审阅、可回溯的 Markdown 笔记。围绕这条主线，本版还带来侧边栏项目拖拽排序、输入框 @Agent 委派，以及一批项目管理、界面自适应与 MCP / embedding 稳定性上的改进。

## 亮点

- **知识空间新增「来源 → 笔记」编译工作流**：现在可以把网页、聊天里的图片 / 音视频、本地文档等学习材料先归档为知识空间的「来源」，经浏览器抓取、图片 OCR、语音转写等方式导入后，由后台整理成可审阅的 Markdown 笔记提案——写入前先查看 diff 再决定是否落盘，生成的笔记还能回溯到对应的来源 Evidence。来源支持刷新与版本化、与外部 vault 同步，以及对相似 / 重复来源的治理。此外新增「来源整理 Agent」设置，可为「来源 → 笔记」这类整理任务单独指定一个更强的 Agent，默认跟随全局默认 Agent，且不影响主对话与 recap 的模型配置。
- **侧边栏项目支持拖拽排序**：可在对话列表里直接拖动调整活跃项目的先后顺序并持久保存，不再只能按更新时间自动排列；开始拖动时会自动收起已展开的项目会话分组，方便整行重排。
- **输入框支持 @Agent 委派子智能体**：在对话输入框输入 `@` 时，Agent 候选会排在文件 / 知识笔记 / 技能之后；选中后会插入一枚可视化 Agent 芯片，并在发送时把「希望某个 Agent 来执行该任务」的意图注入给模型，便于模型通过既有的子智能体工具发起委派。
- **内置模型与语音转写刷新**：补齐 Anthropic / OpenAI / Google Gemini / Moonshot / 智谱 / MiniMax 等供应商的最新旗舰模型（Claude Fable 5、GPT-5.6、Gemini 3.5 Flash、GLM-5.2、MiniMax M3 等），并新增 OpenCode Zen 网关模板；语音转写新增 ElevenLabs Scribe、xAI Grok STT 两个云端后端与 Groq、Mistral Voxtral、DeepInfra 三个 OpenAI 兼容预设，本地 embedding 目录也补充了 mxbai-embed-large。

## 项目与界面

- **项目工作目录选择支持新建目录**：新建 / 编辑项目时，工作目录选择被提升到表单顶部，可选择已有目录或直接新建目录；未手动选择时仍沿用默认项目目录自动创建逻辑。HTTP Server 模式下的远端目录选择器同样支持新建目录，并继续受 `filesystem.allowRemoteWrites` 写入开关保护。
- **项目图标统一为 Logo / 默认文件夹**：新建 / 编辑项目移除了 Emoji 字段，项目列表、快捷指令、定时任务与系统提示统一只用项目 Logo 或默认文件夹图标展示身份；未设置 Logo 的项目也恢复了未读消息角标的正常显示。
- **侧边栏更紧凑，主窗口最小高度统一**：图标侧边栏在高度不足时会自动收起底部的设置快捷入口（仍保留功能区与状态 / 日志区之间的分隔），Plan 历史入口移到日志上方；主窗口最小高度统一为 520，Canvas / Plan 面板不再把窗口挤压到更矮的高度。
- **输入框工具栏自适应与模型名提示**：输入框变窄时会按优先级依次把沙箱、权限模式收进「+」菜单（可就地展开切换），停止 / 发送按钮始终保持在同一行，最窄仅保留「加号 · 模型选择 · 停止 / 发送」；底部工具栏的模型名称过长被截断时，鼠标悬浮即可查看「供应商 · 完整模型名」。

## 修复与稳定性

- **MCP SSE 传输恢复可用**：`kind: sse` 的 MCP 服务器不再被错误地按 Streamable HTTP 直连（此前会报 `400 session_id is required`），改为正确完成 SSE 握手；连接本地地址（`localhost` / 私网）时会自动绕过系统代理，避免「开代理连云端模型」时本地 MCP 被代理劫持。
- **MCP 配置编辑与热更新更可靠**：编辑 MCP server 不再把界面里的 `<redacted>` 占位值写回覆盖真实凭据，服务名与 server id 保持不可变；禁用 / 重排 / 全局开关变更会即时同步到运行时，工具名清洗后重名不再互相覆盖，并发冷启动同一 server 也不会重复握手。
- **技能可读取到已配置的 embedding**：`ha-settings` 技能的 `get_settings("embedding")` 不再返回全 null，而是反映设置中实际启用的 embedding 模型（API 密钥脱敏显示）；embedding 的写入统一改由设置界面管理。
- **消息可只复制选中内容**：在 AI 回复里拖拽选中部分文字后，右键复制与 `Ctrl/Cmd+C` 现在只复制所选文本而非整条消息；未选中时右键菜单仍可一键复制整条消息。
- **消息吸顶与耗时显示更准确**：用户消息悬浮吸顶会按当前可见的回复体切换对应消息，并正确处理折叠行、上下文压缩事件与长会话 DOM 裁剪；工具 / 思考耗时不再把同一轮的耗时重复累加。
