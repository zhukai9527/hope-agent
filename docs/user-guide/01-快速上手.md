# 01 · 快速上手

本章帮你在几分钟内装好 Hope Agent、完成首次配置、开始第一次对话,并了解三种运行模式、如何更新、如何从其他设备远程访问。

> 想配置模型和 API Key 的细节,见 [02 模型与 Provider](02-模型与Provider.md);想用完全免联网的本地模型,见该章的「本地模型一键安装」一节。

---

## 1.1 安装

各平台的完整安装包见 [GitHub Releases](https://github.com/shiwenwen/hope-agent/releases)。下面按平台给出推荐方式。

### macOS(支持最完整)

**方式一:Homebrew(推荐)**

```bash
brew tap shiwenwen/hope-agent
brew install --cask hope-agent
```

> 已经手动装过 `Hope Agent.app`?在 `brew install` 后面加 `--adopt`(接管同版本现有应用,不重新下载)或 `--force`(强制重下覆盖)。

**方式二:手动安装 DMG**

到 Releases 下载 `Hope.Agent_*.dmg`,拖到「应用程序」即可。Apple Silicon 与 Intel Mac 都有原生构建(arm64 / x64),Homebrew 和手动下载都会按你的硬件自动选对版本。

> **若启动时提示「已损坏」或「无法验证开发者」**,在终端执行以下两行后重新打开:
>
> ```bash
> sudo xattr -cr /Applications/Hope\ Agent.app
> sudo codesign --force --deep --sign - /Applications/Hope\ Agent.app
> ```
>
> 这是因为应用采用了 ad-hoc 签名,并非应用本身有问题。

### Windows(实验性)

**方式一:Scoop(推荐)**

```powershell
scoop bucket add hope-agent https://github.com/shiwenwen/scoop-hope-agent
scoop install hope-agent
```

**方式二:手动安装**

到 Releases 下载 `Hope.Agent_*-setup.exe` 双击安装。当前仅提供 x64 版本。

> 若启动时提示缺少 `MSVCP140_1.dll` / `VCRUNTIME140.dll` / `MSVCP140.dll`,请安装 [Microsoft Visual C++ 2015–2022 运行库(x64)](https://aka.ms/vs/17/release/vc_redist.x64.exe) 后重启应用。
>
> Windows 端尚未经过充分测试,欢迎反馈问题。

### Linux(实验性)

提供 amd64(x86_64)与 arm64(aarch64)两种原生构建,覆盖普通 PC、树莓派 4/5、Apple Silicon 跑 Asahi Linux、Graviton / Ampere 云主机。

- **Arch / Manjaro**:`yay -S hope-agent-bin`(或 paru / 任意 AUR helper)
- **Debian / Ubuntu(apt)**:
  ```bash
  curl -fsSL https://repo.hopeagent.ai/pubkey.gpg | \
    sudo gpg --dearmor -o /usr/share/keyrings/hope-agent.gpg
  echo "deb [signed-by=/usr/share/keyrings/hope-agent.gpg] https://repo.hopeagent.ai/apt stable main" | \
    sudo tee /etc/apt/sources.list.d/hope-agent.list
  sudo apt update && sudo apt install hope-agent
  ```
- **Fedora / RHEL / CentOS(dnf / yum)**:
  ```bash
  sudo curl -fsSL https://repo.hopeagent.ai/rpm/hope-agent.repo \
    -o /etc/yum.repos.d/hope-agent.repo
  sudo dnf install hope-agent
  ```
- **openSUSE**:`sudo zypper addrepo …/rpm/hope-agent.repo && sudo zypper install hope-agent`
- **手动**:下载 `.AppImage`(`chmod +x` 后运行)、`.deb`(`sudo dpkg -i`)或 `.rpm`(`sudo rpm -i`),包名含架构后缀,按机器选 `_amd64` / `_arm64`。

## 1.2 首次启动与配置

第一次打开 Hope Agent 会进入**引导向导**。主线只有一句话——**选 Provider 模板 → 填 API Key / Codex OAuth 登录 → 开聊**;其余步骤都可跳过或事后在设置里调整。

桌面版向导实际是 6 步(每步右上角 X 可退出,草稿会自动保存,下次从上次步骤续上):

1. **欢迎**——选界面语言(默认跟随系统,支持 10+ 种语言)和主题(自动 / 浅色 / 深色),切换即时生效。页面底部还有一个可折叠的「连接远程服务器」入口(见 [1.4](#14-从手机或另一台电脑访问))。
2. **模型 Provider(必配)**——从内置的 40+ 个模板里选一个服务(Anthropic、OpenAI、DeepSeek、通义千问等),填 API Key;用 Codex 可走 OAuth 直接登录账号,无需手填 Key。这一步也内嵌了「本地模型一键安装」卡片。跳过按钮会标红提醒。
3. **搜索服务(可选)**——配置网页搜索(DuckDuckGo / Tavily / Brave / Google 等),可跳过。
4. **用户画像(可选)**——填用户名、时区、AI 使用经验、回复偏好,帮助 AI 更贴合你。
5. **审批与安全**——一个开关决定是否开启工具审批。开启(默认)= 危险操作前弹窗确认;关闭 = 所有工具自动放行(相当于全自动模式)。
6. **IM 渠道(可选)**——展示 Telegram / 飞书 / Slack 等渠道卡片,可就地添加账号,也可以之后再去设置里配。点「完成」即结束向导,进入对话。

> - 不想用云端模型?第 2 步里就有**本地模型一键安装**,不需要账号、API Key 或终端。详见 [02 模型与 Provider · 本地模型](02-模型与Provider.md)。
> - 想重新走一遍向导?命令行运行 `hope-agent server setup`(默认即重放向导;加 `--reset` 会先清除已有的「引导完成」状态,已配好的 Provider / 用户配置不会被删)。

配置、会话、记忆等所有数据都会保存在你本机的 `~/.hope-agent/` 目录下(可用环境变量 `HA_DATA_DIR` 改位置)。

## 1.3 三种运行模式

Hope Agent 的三种运行模式**共用同一套核心逻辑**,配置、会话、记忆全部落在 `~/.hope-agent/`,可以随时切换。

| 模式 | 启动方式 | 适合场景 |
| --- | --- | --- |
| **桌面 GUI** | 双击应用图标,或终端 `hope-agent` / `open -a "Hope Agent"` | 功能最全的入口:完整原生 GUI,并内嵌 HTTP/WS 服务——桌面在用的同时也能对外提供接入 |
| **Server + Web GUI** | `hope-agent server start`;`hope-agent server install` 可注册成开机自启的系统服务 | 无窗口守护进程,24 小时在线,IM 渠道 / 定时任务不断线;完整网页版内嵌在服务里,浏览器打开即用,手机 / 平板 / 任意电脑都能直连 |
| **ACP(IDE 集成)** | `hope-agent acp` | 让兼容 ACP 协议的编辑器把 Hope Agent 当作 AI 后端来调用 |

**各平台的启动入口:**

- **桌面 GUI**:Launchpad / 开始菜单 / 应用菜单点图标,或终端运行 `hope-agent`。
- **浏览器 Web GUI**:打开桌面应用后访问 <http://127.0.0.1:8420>;也可以只运行 `hope-agent server start` 启动服务而不打开桌面窗口。
- **ACP**:`hope-agent acp`。

### Server 模式的常用子命令

```bash
hope-agent server start        # 前台启动 HTTP/WS 服务
hope-agent server install      # 注册系统服务(macOS launchd / Linux systemd),开机自启
hope-agent server status       # 查看运行状态
hope-agent server stop         # 停止服务
hope-agent server uninstall    # 卸载系统服务
hope-agent server setup        # 重新运行配置向导(默认即重放;--reset 先清「引导完成」状态,不删 Provider)
```

## 1.4 从手机或另一台电脑访问

如果想在手机、平板或另一台电脑上访问运行在你主力设备上的 Hope Agent:

1. 打开「**设置 → 服务器**」,**设置一个 API Key**(用于鉴权,务必设置)。
2. 把监听地址从 `127.0.0.1:8420` 改为 `0.0.0.0:8420`(允许局域网访问)。
3. 重启后,在其他设备浏览器访问 `http://<运行 Hope Agent 的设备 IP>:8420`,输入 API Key 即可。

> ⚠️ **安全提醒**:不要在没有鉴权(未设 API Key)的情况下把端口暴露到局域网或公网。需要公网访问时,请在前面加一层 HTTPS 反向代理,详见 [Docker 部署指南](../deployment/docker.md)。

## 1.5 自托管(Docker)

想把 Hope Agent 跑在家用 NAS、VPS 或 homelab 上、用浏览器访问网页版:

```bash
docker run -d \
  --name hope-agent \
  -p 127.0.0.1:8420:8420 \
  -v hope-data:/data \
  ghcr.io/shiwenwen/hope-agent:latest
```

容器起来后浏览器打开 <http://127.0.0.1:8420>,按引导向导配置 Provider API Key。镜像覆盖 `linux/amd64` + `linux/arm64`(含 Apple Silicon / 树莓派),随每次发版自动构建。

需要用 docker compose、配合 Ollama 本地 LLM、暴露到局域网、反向代理与 TLS、升级流程,见完整的 [Docker 部署指南](../deployment/docker.md)。也可以一条命令带上本地 LLM 边车:

```bash
docker compose --profile with-ollama up -d    # 起 hope-agent + Ollama 本地模型
```

## 1.6 更新到新版本

桌面应用内置了基于 GitHub Releases 的自动更新,更新方式有三种,任选其一:

1. **应用内检查**:打开「**设置 → 关于**」,点检查更新,发现新版本可一键安装。
2. **对话里说一句**:直接在对话里说「升级」或「检查更新」,AI 会通过 `app_update` 工具帮你检查、确认后安装。
3. **等应用提示**:有新版本时应用会提示你。

> - 通过 Homebrew / AUR / Scoop 安装的版本**同样走应用内置的更新器**;包管理器视角看到的版本号会停留在初装时的值,不影响功能,属正常现象。
> - 更新过程会先校验签名、下载完成后做一次冷启动自检,失败会自动回滚,尽量保证不会更新成一个打不开的版本。
> - 自动更新的行为(是否后台预下载、是否需要手动确认安装)可以在「设置 → 关于 / 自动更新」里调整。桌面版**绝不会在未经你确认的情况下自动重启替换**。
