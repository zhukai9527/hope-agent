# 06 · 设计空间

设计空间让你和 AI 协作,**从一句话或一张参考图产出可交付的设计产物**——网页、移动原型、演示文稿、仪表盘、海报、文档、邮件、图像、动效、音频,乃至真正可交互的组件。生成过程实时预览,可视化微调、版本管理、多格式导出,还能一路推到真实代码工程。

## 6.1 支持哪些产物类型

共 11 种形态,大多是自包含 HTML(浏览器直接渲染,启动快、不白屏);仅交互组件(component)由后端编译成静态产物:

| 类型 | 说明 |
| --- | --- |
| 网页(web) | 落地页 / 桌面网页原型 |
| 移动原型(mobile) | 带设备框和状态栏 |
| 演示文稿(deck) | 自带翻页器,一份文件多页 |
| 数据仪表盘(dashboard) | 网格布局的数据看板 |
| 海报 / 社媒图(poster) | 尺寸预设(方图 / 竖图 / A4 等) |
| 文档 / 报告(document) | 带目录的排版阅读容器 |
| 营销邮件(email) | 兼容邮件客户端的排版 |
| 图像(image) | 用媒体生成模型产出的栅格图 |
| 动效(motion) | 自包含 HTML 动效 |
| 音频(audio) | 文字转语音 / 音乐 / 音效,内嵌播放器 |
| 交互组件(component) | 真正可交互的 React 组件 / 小应用,后端编译成静态产物 |

> **音频**与**图像**产物都需要先配置[媒体生成服务商](02-模型与Provider.md#212-ai-画图与音频生成)(设置 → 模型配置 → **媒体生成模型**,一个服务商填一次 Key 可挂多个图 / 音模型);没有可用模型时生成对话框会给出空态引导,一键跳到配置页。交互组件即使编译失败也会降级成静态错误页,不会白屏。

## 6.2 怎么生成

设计空间首页是一个大输入框,一句话就能开始,不用先建项目。

- **一句话生成**:输入描述,`Cmd/Ctrl+Enter` 生成。可选产物类型、模板快选、内联设计系统和视觉模型。
- **照着参考图做**:首页可以多选 / 粘贴 / 拖拽**最多 5 张**参考图,有图无文也能生成。视觉模型会**直接看原图**,精确还原配色、布局和字体质感。如果当前模型不认图,会自动切到可用的视觉模型并提示;一个视觉模型都没有时会拦下来引导你去设置。
- **从网址提取**:抓取一个网页及其首屏截图,反向提取出它的品牌设计系统。
- **从 Figma 导入**:读取 Figma 里已发布的颜色 / 文字 / 效果样式,蒸馏成设计系统(**仅桌面 / 所有者界面可用**,Figma 令牌按次输入、不落盘、不进模型)。
- **从现有代码工程提取**:从本地代码工程的 CSS / Tailwind 配置 / Design Token 文件归纳出设计系统。
- **品牌包**:一次生成多件带参考图的产物。
- **生成图像 / 音频产物**:选「图像」或「音频」形态时会弹出生成对话框——图像可选宽高比与分辨率,音频可选类型(语音 / 音乐 / 音效)、音色和时长,可选项按当前模型的真实能力显示,并提示将使用哪个服务商与模型。该功能没有可用模型时给出空态引导,一键跳到设置里配置媒体生成模型。

生成是**真流式**的,边生成边成形预览。失败会降级成一个干净的占位页,不会中断,你可以继续在对话里细化。

## 6.3 预览、产物库与工作区

- **单产物预览**:一个稳定的预览窗 + 缩放下拉(纯 CSS 缩放)。**刻意不做无限画布**——那是旧版卡顿的根源。
- **产物库(缩略图墙)**:所有产物的缩略图墙,可跨项目 / 项目内切换查看。缩略图是产物的真实静态预览(不跑脚本,不卡顿)。
- **标签条**:工作区顶部只列你「已打开」的产物,标签可关闭(仅移出,不删文件)、双击改名、拖动排序。真正的**永久删除**则收进右键菜单和产物库墙里(不可逆)。
- **文件夹分组**:产物可以归入文件夹,支持拖拽移动、改名(删文件夹只把产物退回根,不删产物)。
- **设备预览**:可切换 `自动 / 桌面 / 平板 / 手机` 视口;还有全屏「演示模式」。

## 6.4 用对话改产物

工作区左栏内嵌一个 **AI 对话面板**,这是迭代产物的**头号入口**:用自然语言让 AI 改,产物就地落一个新版本,预览自动刷新。

- 每个设计项目有独立的对话线程(从主会话列表和全局搜索中隐藏)。
- 面板每一轮都会自动带上「你当前打开的那个产物」作为上下文,所以你说「把**这个**改成深色版」「给它出个变体」时,改动会落到你正看的产物上。
- 空白时有起步卡引导;AI 回复后输入框上方会出「更精致 / 深色版 / 出个变体 / 质量评审」等下一步建议(点击填入,不自动发送)。

## 6.5 可视化微调与批注

> 这些能力**只对支持元素级微调的 8 种形态生效**(图像、音频、交互组件除外)。

**可视化微调**:点选产物内任意元素,右侧检视器会显示 8 组控件——文本、颜色、排版、间距圆角、布局、尺寸、描边、效果(阴影 / 透明度)。改动**零延迟预览**,确认后自动写回源码并生成新版本;文本可双击就地编辑。还能改链接、上传替换图片(自动内嵌保持产物自包含)。

**撤销 / 重做**:`Cmd/Ctrl+Z`、`Cmd/Ctrl+Shift+Z`,最多 50 步。

**批注**有两种:

- **元素批注钉**——在元素上落一个批注钉(设计变化后自动重新锚定,不丢);工具栏会显示未解决批注的数量角标。
- **画框 / 涂鸦批注**——在预览上框选或圈出区域加说明。

批注可以「一键精修」(让 AI 按批注就地改)或「带到对话」(作为引用塞进输入框,补充后再发)。

## 6.6 版本历史

每次 AI 生成 / 精修、可视化微调、换设计系统、回滚,都会产生一个版本快照。

**版本历史**是双栏的:左栏是版本列表(带「AI 生成 / 手动微调 / 回滚」标记、标题、时间、搜索),右栏是选中版本的实时预览。**恢复**某个历史版本时,会基于它生成一个**新版本**(原历史不动)。版本上限默认 50,超限时优先淘汰最旧的手动微调版本,保留 AI 生成和回滚这些里程碑。

## 6.7 导出

导出优先用真实浏览器原生捕获(最高保真),缺条件时自动回退到客户端方案,**始终能导出**:

| 格式 | 用途 |
| --- | --- |
| HTML | 自包含单文件,可直接分享 / 打开 |
| PNG | 全保真截图(可选 1/2/3 倍清晰度) |
| PDF | 矢量、文字可选可搜(演示文稿导出为每页一张) |
| 视频 MP4 | 动效 / 交互产物录制 |
| PPTX | 演示文稿 |
| ZIP | 打包交付 |
| Markdown | 低带宽交付 |
| 代码交付包 | 含 HTML + 源码 + 六种格式 Token 代码 + 交付说明,给工程侧 |

> 视频和高保真 PDF/PNG 需要 Chromium 和 ffmpeg——它们**不打进安装包**,系统已装的会直接用,缺失时按需下载,任何缺失都自动降级到客户端方案,不会卡死。桌面版导出弹原生「保存到…」框;网页 / 服务器模式走浏览器下载,**远端不写服务器磁盘**。

## 6.8 设计系统与 Design Token

设计系统是一份**可复用的品牌契约**——一份 `DESIGN.md`(涵盖品牌、色彩、字体、间距、布局、组件、动效、语气、禁忌 9 段)加一张 Token 表(`--ds-*` CSS 变量)。套用到产物上就是「换皮」,一致性由 Token 锁定。

**两类内置系统**:

- **6 套原创原型语言**——极简现代、编辑杂志、科技暗色、温暖亲和、专业金融、大胆活力,覆盖常见气质。
- **一批品牌风格参考**——覆盖主流品牌风格。**这些都是对各品牌公开视觉语言的独立再诠释、仅供设计参考**,预览 / 渲染时会自动附一行免责声明(非官方、无隶属 / 授权,商标归各自所有者)。原创系统不附免责声明。

你可以**反向提取**(从截图 / 图片 / 网址 / 代码工程 / Figma 一键提取设计系统)、**可视化调 Token**(逐个变量手调,颜色给取色器)、把 Token 一键导出为 **CSS / SCSS / TypeScript / Swift(iOS) / Android XML / DTCG** 六种开发者格式,也可以导入 / 导出 `DESIGN.md` 作为跨工具互通格式。

## 6.9 交付到代码工程

设计空间可以一路推到真实代码:

- **绑定代码仓库**——设计项目可以绑定到一个本机目录或一个 Hope Agent 项目(**仅桌面 / 所有者界面可设**,AI 不能自授权)。
- **实现到代码**——产物导出菜单里的「实现到代码…」把设计稿交给一个普通聊天会话,在绑定的仓库里落地实现,复用完整的[权限审批](07-工具与权限.md)、Diff 面板、[计划模式](08-自主任务.md#85-计划模式plan-mode)和[隔离工作树](07-工具与权限.md#79-文件操作git-与隔离工作树),写代码每一笔都走审批。
- **代码变化回灌**——实现落地后,如果代码侧又发生了会话之外的改动,设计空间会检测到「漂移」并在产物上打标记,提示你「查看代码变更 / 带到设计对话 / 标为已同步」。
- **同步设计系统到代码**——把设计系统绑到代码目录,一键把六种格式 Token 文件写进去(改了 Token 重新同步,代码即时更新)。
- **只读分享 / 部署**——服务器模式可生成只读分享链接;可选一键部署到 Cloudflare Pages 或 Vercel(需手动开启,凭据仅在界面里配置)。

> **需求不清时,设计 AI 会先问**:在对话里弹结构化的发现问卷,或需要定视觉方向时弹「视觉风格卡」(每个选项带调色板、字体样张和气质描述),选定后作为该产物的设计系统。需求写清楚了就直接开做,不啰嗦。

## 6.10 设置

**入口**:设置 → 设计空间(整个分类为中风险,因含有会产生模型调用成本的「自动质量评审」)。

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 启用设计空间 | 开 | 全局开关 |
| 生成后自动聚焦预览 | 开 | 生成产物后是否自动打开预览 |
| 默认设计系统 | 无 | 新产物在项目未指定时的回退设计系统 |
| 定稿前自动跑质量评审 | **关** | 对产物做 5 维质量评审(品牌契合 / 可访问性 / 视觉层次 / 可用性 / 性能),**会产生一次模型调用成本** |
| 反 AI-slop 自查 | 开 | 无模型成本的确定性检查,抓近乎空壳 / 占位的内容并标「待复查」 |
| 每个产物保留的版本数 | 50 | 版本上限 |
| 导出清晰度 / PDF 质量 | 2 倍 / 92 | 导出栅格化倍率与压缩质量 |

> **无痕会话里设计空间完全不可用**、零留痕(无痕会话本身与项目 / IM 渠道互斥)。

## 附:Canvas 与设计空间的区别

你在**主对话**里可能会遇到 **Canvas**——那是对话内随手出图的轻量沙盒,AI 在聊天中直接创建一个可视化(HTML / Markdown / 代码 / SVG / Mermaid / 图表 / 幻灯片),在右侧面板预览。它适合「顺手画一下」,不做长期管理。

而**设计空间**是成体系的创作工作空间,有产物库、版本、导出、设计系统和代码绑定。两者相互独立,各司其职:随手可视化用 Canvas,正经做设计用设计空间。
