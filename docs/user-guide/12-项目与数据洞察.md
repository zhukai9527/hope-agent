## 12.1 项目容器

项目把多个相关会话聚成一个工作空间,共享项目指令、项目记忆和统一的工作目录。项目是**可选的**——不归项目的会话完全不受影响。

**创建**:侧边栏项目树 → 新建项目,填名称、描述、图标、颜色、默认 Agent、工作目录,以及项目指令。

**核心特性**:

- **工作目录 = 真实文件夹**:你显式选的目录,或默认的 `~/.hope-agent/projects/{项目}/workspace/`。上传的文件直接落到这个目录里,AI 通过工作目录的顶层文件清单和 `read` 工具感知它们(没有单独的文件表)。
- **项目指令**:根目录的 `AGENTS.md`,可直接编辑,会装配进这个项目下每个会话的系统提示词。
- **项目记忆**:优先级最高(项目 > Agent > 全局),详见 [04 · 三层记忆](04-记忆系统.md#41-三层记忆全局--agent--项目)。
- **懒创建会话**:进项目「新建对话」不会预先建空会话,停在草稿态,发第一条消息才落库。项目会话与无痕互斥。
- **把 IM 会话归项目**:在该 IM 聊天里用 `/project <id>`,直接把现有会话归入项目(不新建会话)。

**项目设置面板**有四个标签:概览(状态 + 最近会话)、文件(文件浏览器,可上传 / 删除 / 重命名)、指令(编辑 `AGENTS.md`)、自动记忆(管理项目自动记忆)。还可以绑定知识空间。

> **删除项目**会依次:解绑会话(会话本身保留)→ 删项目 → 删项目目录 → 删项目记忆。**你显式选的外部目录永远不会被删**。

## 12.2 自定义 Agent

每个 Agent 是磁盘上的一份配置加若干 Markdown,定义它是谁、用哪个模型、有哪些能力。你可以创建多个 Agent(比如一个「编程助手」、一个「写作助手」),互不干扰。

**入口**:设置 → Agent。编辑页有 8 个标签:

| 标签 | 内容 |
| --- | --- |
| 身份 | 名称、描述、emoji、头像、行为说明 |
| 性格 | 角色 / 氛围 / 语气 / 原则 / 边界,或自由文本 |
| OpenClaw 兼容模式 | 兼容 OpenClaw 格式的原始 Markdown 人格编辑 |
| 工具 & 技能 | 工具 / 技能开关、沙箱、审批、异步策略、MCP 开关、**新会话的默认权限模式** |
| 模型 | 主模型 / 备用链 / Plan 专用模型 / 温度 / 思考强度(会话 > Agent > 全局) |
| 记忆 | 该 Agent 的记忆开关与预算 |
| 子 Agent 调用 | 见 [09 · 子 Agent](09-多Agent与定时任务.md#91-子-agent) |
| 审批 | 自定义需审批的工具清单 |

**新会话用哪个 Agent** 按固定优先级决定(第一个非空的胜出):显式参数 → 项目默认 → IM 话题 / 群 / 频道 / 账号绑定 → **全局默认 Agent** → 内置主 Agent。所以你可以在三个层面设默认 Agent:

- **全局**:设置 → Agent(在「默认 Agent」区设置)。
- **项目级**:项目设置里的默认 Agent。
- **IM 账号级**:渠道账号里的绑定 Agent。

> 内置主 Agent 不可禁用 / 删除。删除某个 Agent 时需要指定一个替代 Agent(引用会重绑),且是可恢复删除(移入回收站)。

## 12.3 Dashboard 数据大盘

Dashboard 跨会话、日志、定时任务聚合分析,用图表展示成本、Token、活跃度、健康度和任务执行情况。

**入口**:侧边栏的 Dashboard,有多个标签。主要维度包括:

- **综合概览**:会话数、消息数、输入 / 输出 Token、工具调用、错误、活跃 Agent、活跃定时任务、估算成本、平均响应时间。
- **Token 用量趋势**:按天 / 模型 / 调用类型 / 用途 / 领域分组。
- **工具使用、会话趋势、错误趋势**。
- **洞察(Insights)**:同环比、成本曲线、活跃热力图、Top 会话、模型性价比、四维健康度评分。
- **系统指标、本地模型、学习记录**。
- **目标与执行大盘**:目标验收率、工作流完成率、持续推进强进展率、任务与计划明细。
- **能力评测**:用真实模型运行核心 Agent 合成场景,查看任务完成、工具、耗时、Token、费用、对比与趋势。完整操作见 [14 · 能力评测](14-能力评测.md)。

**模型用量总账**:所有会触发模型推理的调用(对话、后台查询、摘要、Embedding、语音、判官、搜索、生图、生音、视觉等)都会统一记账,Dashboard 的 Token / 成本总量以此为准。本地模型、语音、Embedding 这些不返回 token 的调用只记次数和耗时,**不会用字符估算冒充准确 token**。

> 会话级统计自动排除定时任务、子 Agent 和无痕会话;**无痕会话绝不入账**。成本估算优先用你配置的模型单价(人民币计价会自动换算),否则用内置定价表兜底。

## 12.4 Recap 复盘报告

Recap 对选定时间段的会话做深度分析,生成一份多章节的复盘报告,可导出为独立 HTML 分享。

**入口**:Dashboard 的 Recap 标签,或 `/recap` 命令。可选范围(增量 / 7 天 / 30 天 / 90 天)。

**报告内容**:多个 AI 分析章节(项目领域、交互风格、有效工作流、摩擦分析、优化建议、记忆 / 技能推荐、成本优化、功能建议、未来探索等)加上 Dashboard 的量化指标和汇总图表。**导出的 HTML 是自包含单文件**(内联全部样式,零外部依赖),支持明暗双主题,可直接分享。

**设置(设置 → 复盘报告)**:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 分析模型链 | 继承全局 | 复盘分析用哪些模型(与主对话解耦,支持跨模型降级) |
| 输出语言 | 跟随界面 | 报告语言 |
| 默认范围 | 30 天 | 无历史报告时的默认范围 |
| 单次最大会话数 | 500 | 上限 |
| 缓存保留天数 | 180 | 报告缓存保留期 |
