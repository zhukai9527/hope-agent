## 13.2 用对话改设置

除了在界面里点,你还可以**直接用对话让 AI 帮你改设置**——比如「把主题改成深色」「把工具超时设成 60 秒」「关掉通知」。AI 通过内置能力读写设置,**永远不会直接编辑配置文件**。

按风险分三档确认:

- **低风险**(外观 / 显示偏好):一句话说明就改。
- **中风险**(影响上下文 / 成本 / 输出质量,如压缩、记忆、搜索、审批):先给你看「当前值 → 新值」再改。
- **高风险**(安全 / 网络暴露 / 凭据 / 权限规则 / 需重启,如代理、全局快捷键、危险模式、服务器、自动更新、保护路径等):**必须你明确确认后才改**。

**有几类设置 AI 只能读、不能改,只能你在界面里改**(出于凭据安全和运行稳定):

- **Provider 列表与 API Key**
- **媒体生成服务商与 API Key**(默认模型链和生成参数仍可让 AI 改)
- **IM 渠道账号**
- **MCP 服务器配置**
- **主 / 备用模型的选择**
- **Embedding(向量)模型的选择**
- **语音识别(STT)服务商**
- **Hooks 钩子配置**

即使是 AI 能读的设置,凭据字段(API Key 等)在读取时也会自动**打码**,不会出现在对话里。

## 13.3 配置备份与一键回滚

每次改配置前,系统都会自动留一份旧配置副本,改坏了随时能回退,而且**回退动作本身也可以再撤销**。

- **自动快照**:每次配置写盘前自动留档,保留最近 50 份,文件名记录了「谁在何时改了什么」。
- **全量崩溃备份**:崩溃诊断触发或你手动触发时,把整套配置目录打包,保留最近 5 份。
- **恢复默认**:每个设置面板标题栏都有一个恢复控件,可以恢复该页 / 该区域的默认值。
- 你也可以在对话里让 AI 列出备份、回滚到某个快照(这属于高风险,会先给你看时间戳和内容再确认)。

配置变更后界面会自动热更新(主题、语言、通知等),大多数无需重启。

## 13.4 数据与安全

**本地优先**——所有数据(会话、记忆、技能、配置、凭据)都存在你本机的 `~/.hope-agent/` 目录里,不上传云端。API Key 只用于**直接调用你配置的 Provider**,Hope Agent 不做任何中转。

| 主题 | 说明 |
| --- | --- |
| **数据自有** | 数据全在 `~/.hope-agent/`;[无痕会话](03-对话与会话.md#36-无痕会话)「关闭即焚」,不进侧栏 / 搜索 / 统计 |
| **API Key 直连** | 密钥只用于直连你配置的模型服务,不经第三方 |
| **日志绝不含密钥** | 所有日志强制脱敏,API Key / OAuth Token 禁止出现在任何日志里 |
| **服务模式鉴权** | HTTP/WS 服务用 Bearer Token 鉴权(`/api/health` 免鉴权);绑 `0.0.0.0` 对外暴露前**务必设 API Key** |
| **SSRF 防护** | 所有出站请求强制安全检查,默认拦截访问内网和云元数据地址;可在设置 → 安全策略里调整策略、加可信主机白名单 |
| **凭据存放** | OAuth Token 等凭据存在 `~/.hope-agent/credentials/`;登出时会清除 |

**危险模式(全局 YOLO)** 是一键跳过全部工具审批的开关(设置 → 安全策略,或命令行 `--dangerously-skip-all-approvals`),**极高风险,只在完全信任的本地环境用**。它与计划模式正交——只跳审批,不解除计划模式的限制。详见 [07 · 全局 YOLO](07-工具与权限.md#75-全自动超时与无人值守)。

## 13.5 可靠性:保活与崩溃恢复

Hope Agent 用多层冗余保证长期运行不掉线:

- **Guardian 父子进程**(桌面版):一个监工进程守护主程序,崩溃后自动退避重启。
- **系统服务保活**:`hope-agent server install` 后交给操作系统(launchd / systemd)拉起。
- **子系统自愈**:MCP 断连自动重连、IM 各账号独立重连、定时任务幂等恢复。
- **崩溃自诊断**:连续崩溃到一定次数会自动跑一次诊断——先全量备份,再用最便宜的可用模型分析原因、做安全的就地修复。**自动修复只动配置和明确损坏的数据库,绝不动你的会话、记忆、技能和 Provider 列表**。

**入口**:设置 → 系统健康,可以查看崩溃记录、诊断结论、已应用的修复,以及全量备份的恢复。

## 13.6 问题反馈与自诊断

你可以直接在对话里让 AI 把 bug、需求或改进整理成 GitHub issue 草稿并提交,也可以让它读源码和日志回答「Hope Agent 是怎么实现 X 的」。

- **入口**:设置 → 工具设置 → Issue 上报(配置目标仓库、GitHub Token 等),或对话里直接说。
- **提交前总会二次确认**;标题、正文和证据都经过脱敏;GitHub Token 绝不出现在对话、日志或 issue 里。

## 13.7 其它常用设置

| 设置 | 位置 | 说明 |
| --- | --- | --- |
| **主题** | 通用 → 外观 | 自动 / 浅色 / 深色,热切换 |
| **界面语言** | 通用 → 外观 | 支持十余种语言;注意「界面语言」和「告诉 AI 的回复语言偏好」(在个人资料里)是两回事 |
| **背景动效** | 通用 → 外观 | 全天候背景,可与天气特效联动 |
| **开机自启 / 防止休眠** | 通用 → 系统 | 防休眠会让系统不睡(显示器仍可关) |
| **全局快捷键** | 通用 → 系统 | 默认「快速聊天 = Alt+Space」,可自定义;高风险(可能与系统冲突) |
| **代理** | 通用 → 网络 | 跟随系统 / 直连 / 自定义;影响所有出站请求;高风险 |
| **通知** | 通知 | 总开关、是否含回复预览、后台任务完成提醒等 |
| **天气组件** | 工具设置 → 通用 | 城市 / 定位,与背景动效联动 |
| **个人资料** | 个人资料 | 头像(可裁剪)、姓名、时区、AI 使用经验、回复风格等,帮助 AI 更贴合你 |
