## 2.1 接入模型服务商(Provider)

Hope Agent 通过「**服务商 + API Key**」接入各家大模型。内置 **40+ 个服务商模板、300+ 个预设模型**,也支持自定义任意 OpenAI 兼容或 Anthropic 端点。

**入口**:设置 → **模型配置** → **服务商** 标签页 → 点右上角「**添加服务商**」。

添加向导首页提供五条路径:

1. **用 ChatGPT 登录(Codex)**——顶部主按钮,用账号登录而非 API Key,详见 [2.4](#24-用-chatgpt--codex-账号登录)。
2. **连接远程服务器**——连到一台已经跑着 Hope Agent 服务的机器,复用它的配置,见 [01 · 从手机或另一台电脑访问](01-快速上手.md#14-从手机或另一台电脑访问)。
3. **本地模型助手**——未装 Ollama 时出现,一键安装本地模型,详见 [2.9](#29-本地模型一键安装)。
4. **选择内置模板**——可搜索的服务商网格,已配置的会打绿色「已配置」角标。
5. **自定义服务商**——接入任意兼容端点。

### 用内置模板(推荐)

点一个模板(如 Anthropic、OpenAI、DeepSeek、通义千问),进入配置页填 **API Key**(部分本地 / 无鉴权端点的 Key 标注为「可选」),可点「测试连接」验证,再「完成」保存。模板已经帮你预填好了端点地址、API 类型、思考风格和模型列表,通常不用改。

新增服务商后,系统会自动把它的第一个模型设为当前默认模型,你可以立刻开始对话。

### 自定义服务商

如果你的服务商没有现成模板,选「自定义服务商」走三步向导:

1. 选 **API 类型**(Anthropic / OpenAI Chat / OpenAI Responses,见 [2.3](#23-四种-api-类型));不确定就选 **OpenAI Chat**,兼容性最好。
2. 填服务商名、**Base URL**、API Key(可选)、思考风格,可测试连接。
3. 手动添加模型:模型 ID、显示名、支持的输入类型(文本 / 图片 / 视频)、上下文窗口、最大输出、是否为推理模型、单价。

### 服务商的常用设置

在服务商列表里可以拖拽排序、启用 / 禁用、编辑、删除(右侧「⋮」菜单)。编辑页里还有这些字段:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 计价货币 | 美元(USD) | 模型单价的币种(USD / CNY),只影响成本统计的换算;内置的通义 / 火山 / 腾讯等已标为人民币 |
| 允许访问私有网络 | 关 | 仅当 Base URL 是内网地址(如本地 Ollama / LM Studio)时出现;开启后放行 localhost,并把该主机加入安全白名单 |
| User-Agent | `claude-code/0.1.0` | 请求头里的 UA,一般不用改 |
| 思考风格 | 模板值 | 决定推理参数怎么发,见 [2.6](#26-温度与思考强度) |
| 模型列表 | 模板预设 | 每个模型可配 ID、显示名、输入类型、上下文窗口、最大输出、是否推理、单价 |

> **关于 API Key 的安全**:桌面 GUI 属于本机可信环境,Key 字段是带「眼睛」图标的密码框,可以本地明文查看,编辑时留空表示保留原有 Key。**但 Provider 列表与 API Key 不能通过对话让 AI 修改**——这是刻意的安全设计,AI 读取时看到的是打码后的值。详见 [13 · 设置与安全](13-设置与安全.md)。

## 2.2 一把服务商配多把 API Key(自动轮换)

给同一个服务商挂多把 API Key,某把限流或失效时会**自动切到下一把**,让用量更均衡、更抗限流。

**入口**:**编辑已有服务商**页里的「认证配置 / Auth Profiles」区块。注意——初次接入向导只填单把 Key,多把 Key 需要保存后再回来编辑该服务商添加。

每条 Key 配置包含:标签名(便于区分)、API Key、启用开关,以及可选的独立 Base URL。

**它怎么工作**:遇到限流、过载、鉴权、计费类错误会自动换 Key,并给失败的 Key 一段冷却时间;网络超时这类「换了也没用」的错误则只重试同一把。同一个会话会尽量「黏」在同一把 Key 上,以便命中模型的 prompt 缓存、省成本。

> 用 ChatGPT / Codex 账号登录的服务商不参与 Key 轮换(它没有可轮换的 Key),见 [2.4](#24-用-chatgpt--codex-账号登录)。

## 2.3 四种 API 类型

API 类型告诉 Hope Agent 用哪套协议和服务商通信。**用内置模板时它已经预填好,你不用关心**;只有自定义端点时才需要按服务商文档选对。

| 类型 | 说明 | 什么时候用 |
| --- | --- | --- |
| **OpenAI Chat** | 最通用的 `/v1/chat/completions` 协议 | 绝大多数第三方 / 国内 / 聚合服务商(DeepSeek、通义、智谱、Moonshot、火山、OpenRouter、本地 Ollama 等)。**自定义时的首选** |
| **Anthropic** | Claude 官方的 Messages API | Anthropic 及采用其协议的服务 |
| **OpenAI Responses** | OpenAI 的新协议 `/v1/responses` | OpenAI 官方(默认)、GitHub Copilot |
| **Codex** | 用 ChatGPT 账号登录,而非 API Key | 见 [2.4](#24-用-chatgpt--codex-账号登录);**不能手动选,只能通过登录产生** |

跨协议自动切换模型时(比如主模型失败降级到另一家),系统会自动做消息格式转换,你无需关心。

## 2.4 用 ChatGPT / Codex 账号登录

不想手填 API Key?可以用 ChatGPT 账号直接登录,把 GPT-5.x 系列当主对话模型,消耗你的订阅额度。

**登录**:设置 → 添加服务商 → 首页顶部「**使用 ChatGPT (Codex) 登录**」主按钮,会弹出系统浏览器完成授权(5 分钟内完成)。登录成功后自动保存配置,显示绿色「已配置」角标。

也可以用命令行登录(适合远程 / 无图形界面的机器):

```bash
hope-agent auth codex login          # 本机登录
hope-agent auth codex login --no-open # 只打印授权链接(配合 SSH 端口转发)
hope-agent auth codex status         # 查看登录状态
hope-agent auth codex logout         # 登出(会删除 Codex 服务商与本地凭据)
```

**重新登录**:Codex 服务商卡片的「⋮」菜单里有「重新登录」(而不是「删除」,避免误删);token 过期时界面也会引导你重新登录。

- 登录后使用的具体模型可在设置里切换,默认选择所有账号档位都可用的那一档。
- Codex 凭据只存在本机(`~/.hope-agent/credentials/auth.json`),绝不写入日志。
- Codex 登录与 [11 · MCP 接入外部工具](11-连接与扩展.md#111-mcp接入外部工具) 的账号登录是两套完全独立的机制,互不影响。

## 2.5 选主模型与备用模型链

**入口**:设置 → 模型配置 → **全局模型** 标签页。

- **默认模型**:两级下拉(先选服务商、再选模型),是所有新会话默认使用的模型。
- **备用模型链(Fallback)**:可添加多个、拖拽排序;当主模型调用失败时,按这个顺序自动尝试下一个,详见 [2.7](#27-自动容错failover)。

**几个要点**:

- 会话在创建时会「固定」当时的模型;之后你改全局默认**不会影响已经存在的会话**,只影响新会话。想给单个会话临时换模型,用对话里的 `/model` 命令或输入框的模型入口。
- 每个 Agent 也可以单独配自己的模型链;Agent 没配就跟随全局。
- 备用链只是本轮临时降级,下一轮仍从主模型重新开始。

## 2.6 温度与思考强度

两个影响回答风格的旋钮,都按「**会话 > Agent > 全局**」三层覆盖(会话级优先级最高)。

**入口**:设置 → 全局模型。

- **思考强度(Think)**:控制模型思考的深度,取值最多为 `none / minimal / low / medium / high / xhigh`(具体可选档位随模型 API 类型而定,如 Claude / OpenAI Chat 仅 `none / low / medium / high`),全局默认 `medium`。对话里可用 `/thinking high` 快速调整。
- **温度**:控制回答的随机性,滑块 0.00–2.00,默认 1.00。带「重置」按钮回到继承状态。

**思考风格(Thinking Style)** 是服务商级(也可按模型覆盖)的设置,决定推理参数「怎么发送」,内置模板已预填,通常不用动。共 5 种:

| 风格 | 适用 |
| --- | --- |
| OpenAI(`reasoning_effort`) | OpenAI / DeepSeek / Mistral / xAI 等 |
| Anthropic(`thinking budget_tokens`) | Claude、MiniMax、Kimi 等 |
| Zai | 智谱 Z.AI |
| Qwen(`enable_thinking`) | 通义千问 / 阿里云百炼 |
| None | 不发送思考参数(部分自定义端点) |

> 如果某个模型本身不是推理模型,系统会自动不发送思考参数;你不用手动区分。

## 2.7 自动容错(Failover)

这是内建的能力,**没有独立开关**,跟着你的模型链([2.5](#25-选主模型与备用模型链))和多 Key([2.2](#22-一把服务商配多把-api-key自动轮换))自动生效。任何模型调用失败时,系统会自动判断错误类型并处理,你基本无感,偶尔会看到对话里的「模型已降级」提示。

处理策略随场景不同:

- **主对话**:最多重试、允许换 Key、失败换下一个模型(最强容错)。
- **后台任务**(如生成标题、提取记忆):轻量重试(用户不必久等)。
- **上下文摘要**:快速失败降级(你正在等回复,宁可快)。

常见错误的处理方式:限流 / 过载 → 退避重试 + 换 Key;网络超时 → 只重试;上下文超长 → 自动压缩后重试;鉴权失败 → 换 Key(Codex 则提示重新登录);模型不存在 → 直接换下一个备用模型。

> **Codex 用账号登录,没有多把 Key 可换**,所以它一旦鉴权失败会直接尝试模型链里的下一个模型,并提示你重新登录 Codex。

## 2.8 后台模型与视觉模型

在「全局模型」面板底部,可以为两类特殊用途单独指定模型,独立于主对话模型:

- **视觉桥模型(Vision)**——当你的主模型看不懂图片(不支持视觉)、却收到图片时,用这个模型把图转成文字描述注入对话,替代「直接丢图」。**默认关闭**,需要手动指定一个支持图片的模型才开启。转录的文字会被当作「不可信外部数据」处理,不会被当成对 AI 的指令。
- **自动化默认模型链(Automation)**——为「只需调一次模型、不需要完整 AI 人格」的后台任务(生成会话标题、Recap 报告、记忆整理、知识空间维护、技能评审等)指定统一的默认模型链。各功能面板里若单独设了模型,则优先用它自己的;否则用这里的全局默认;再否则跟随主对话模型。

两者都是纯模型引用、不含凭据,风险等级为中(MEDIUM)。

## 2.9 本地模型一键安装

不需要账号、API Key 或终端,在设置里选一个适合你硬件的本地模型,应用会帮你完成 **Ollama 安装 → 模型下载 → 注册服务商 → 设为默认** 的全流程。数据完全本地,不联网也能用。

### 省心入口:本地模型助手卡片

**位置**:模型设置页,或首次向导的 Provider 步骤里。

卡片会根据你的硬件自动推荐一个模型(显示大小、上下文窗口、是否推理型),可以「展开备选」换其它候选。一个主按钮会随 Ollama 状态自适应:

- 没装 Ollama → 「安装 Ollama」(macOS / Linux 自动安装;Windows 会引导你去 [ollama.com/download](https://ollama.com/download) 下载)。
- 装了没运行 / 推荐模型没下载 → 「安装模型」。
- 都就绪 → 显示「已启用」。

完成后会**自动**把该模型加入服务商列表并设为全局默认。安装 / 下载过程有进度弹窗(阶段、字节、预计剩余时间、日志),可以后台化,也可以取消。

### 显式管理:本地模型标签页

顶部状态区显示 Ollama 状态(未安装 / 已安装 / 运行中),已装模型列表里每个模型都能:启动 / 停止(加载到内存或卸载)、加入服务商、设为默认、加入 Embedding 配置、删除等。还能搜索 Ollama 模型库、手动输入名字下载。

> **推荐依据**:预算取 macOS 统一内存的 60%、或 Windows/Linux 独显显存的 60%(不足则回落系统内存的 60%),再扣约 1 GiB 运行时开销,按预算从大到小选第一个装得下的模型。
>
> **App 不接管 Ollama 进程**:只在需要时尝试启动它,应用退出时**不会杀掉 Ollama**,不影响你其它用到 Ollama 的工具。删除模型前会列出所有引用并让你确认。

## 2.10 记忆向量模型(Embedding)

配置一个 Embedding(向量)模型,可以让[记忆](04-记忆系统.md)和[知识空间](05-知识空间.md)做**语义检索**(按意思找,而不只是按关键词)。不配则退化为纯关键词检索。

**入口**:设置 → 记忆 / Embedding 模型面板。三种配置方式:

1. **本地快捷卡**:一键装 Ollama + 下载推荐向量模型,自动设为默认并重建向量。
2. **模板快速新建**:OpenAI / Google Gemini / Jina / Cohere / SiliconFlow / Voyage / Mistral / Ollama。
3. **自定义**:填服务商类型、Base URL、API Key、模型名、维度。

> **切换默认 Embedding 模型会触发全量重建向量(reembed)**,并需要二次确认——因为不同模型的向量空间不通用。记忆与知识空间共用同一个「模型库」,但各自独立选用哪个。
>
> Embedding 模型选择**只能在 GUI 里改**,不能通过对话让 AI 修改(它携带 API Key,且换模型有重建向量的重副作用)。

## 2.11 语音转写(STT)

独立于主模型的「语音转文字」引擎,用于:桌面麦克风输入(可边说边出字)、以及 IM 渠道里收到的语音消息自动转写。

**入口**:设置 → **语音识别** 面板。

- **主模型**:桌面语音输入首选的转写模型(流式模型会标注 `streaming`,可边说边出字)。
- **IM 兜底模型**:IM 自动转写专用(只能选非流式模型);不设则用主模型。
- **服务商**:「添加服务商」支持 OpenAI、Groq、ElevenLabs、xAI、DashScope(百炼)、Deepgram、AssemblyAI、Azure、讯飞、火山等 10 种协议;本地后端(whisper.cpp / faster-whisper / FunASR / sherpa-onnx)可一键探测接入。

**桌面录音怎么用**:聊天输入框的麦克风按钮点击录音;也支持在应用窗口内按住 `Ctrl+Shift+H` 说话、松开转写,文字插入到光标处。录音时显示波形和实时耗时。

**IM 自动转写**:默认关闭,开关在**每个 IM 渠道账号**的编辑对话框里(不在语音识别面板)。开启后,IM 里收到的语音会自动转成文字(前面加 `[语音转录]` 前缀)注入对话。

> 语音服务商列表与 Key 同样**只能在 GUI 改**,AI 只能读到打码摘要;唯一能用对话调整的是「IM 自动转写」开关。单次音频上限 25 MiB。

## 2.12 AI 画图与音频生成

图片和音频生成共用一套「**服务商 → 多模型 → 功能默认链**」配置,供对话里的 `image_generate` / `audio_generate` 工具和[设计空间](06-设计空间.md)使用。

### 第一步:配置服务商与模型

**入口**:设置 → **模型配置** → **媒体生成模型** 标签页。点「添加服务商」从 27 个内置模板一键接入,模板按模态分组并支持搜索——品牌名和模型 ID 都能搜(例如输入 `seedream` 或 `flux` 直接定位到对应服务商):

- **图像**:OpenAI(gpt-image)、Google(Gemini / Imagen,支持改图和多参考图)、Fal(Flux)、SiliconFlow(Qwen-Image)、智谱(CogView)、通义万相、火山引擎(Seedream)、腾讯混元、阶跃星辰、百度千帆、商汤日日新、Black Forest Labs(FLUX)、Stability AI、Replicate、Together、xAI(Grok Image)、Recraft、快手可灵、科大讯飞
- **音频**:ElevenLabs(TTS + 音乐 + 音效)、MiniMax(语音 + 音乐)、OpenAI(TTS)、Cartesia、Deepgram、Fish Audio、Hume、火山豆包语音、Stability(音效)、快手可灵
- **自建端点**:「自定义(OpenAI 兼容)」

部分服务商的凭据格式需注意:快手可灵与火山豆包语音同时支持新旧两套鉴权,填 `ak:sk`(可灵)或 `app_id:access_key`(豆包语音)即按旧版签发;科大讯飞需填 `appid:apikey:apisecret` 三件套。

和语言模型一样,**一个服务商填一次 Key、挂多个模型**,图像和音频模型可以混挂在同一个服务商下。模板自带的预设模型会带上能力信息(支持的尺寸、宽高比、分辨率、是否支持改图 / 蒙版重绘、音频类型、时长范围、是否需要音色),这些能力决定了后续哪些参数会在界面上亮起。也可以「添加自定义模型」手工填模型 ID 并标注能力。

服务商卡片可拖拽排序——顺序即**自动挑选和失败切换的优先级**。ElevenLabs / OpenAI / Cartesia / MiniMax 可「拉取音色」列出账号可用音色,并设一个服务商级默认音色;其余语音服务商保留手填音色 ID。API Key 在面板里是密码框;AI 通过对话读取时会打码,而且**不能写入服务商条目**。

### 第二步:配置默认调用链与工具参数

**入口**:设置 → **工具设置** → **媒体生成** 标签页。为**图像、语音、音乐、音效**四类各配一条默认调用链(主模型 + 若干备用,主模型失败自动降级到备用)。留空则「跟随服务商顺序(自动)」,下方小字会提示实际会用到哪个模型。

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 图像默认尺寸 | 1024×1024 | 调用未指定时的出图尺寸 |
| 图像默认宽高比 / 分辨率 | 未指定 | 调用未指定时的兜底(模型支持才生效) |
| 图像请求超时 | 180 秒 | 单次生成超时(30–900) |
| 音频默认时长 | 未指定 | 音乐 / 音效未指定时长时的兜底 |
| 音频请求超时 | 300 秒 | 单次生成超时(30–900) |

两个图像 / 音频总开关也在这里,关掉后对应的工具不再出现在 AI 的工具列表里。

### 在对话和设计空间里用

对话里直接说「画一张……」触发 `image_generate`,说「生成一段旁白 / 背景音乐 / 音效」触发 `audio_generate`(可指定语音 / 音乐 / 音效、音色和时长)。有参考图时,能力数据里声明支持改图的模型会自动切到对应的改图端点;蒙版重绘只会投给声明支持蒙版的模型,做不到的候选会被跳过、自动换下一个。

[设计空间](06-设计空间.md)的图像 / 音频产物走同一套配置,生成对话框里可直接选宽高比、分辨率、音频类型、音色和时长(按模型能力显示)。**还没配任何服务商时**,这些入口会显示引导卡片,点一下就跳到上面第一步的设置页。

> 通过对话调设置时,AI 只能改默认调用链和上表里的参数;服务商条目(含 API Key)**只能在 GUI 里改**。

## 2.13 Web 搜索与网页抓取

### Web 搜索(web_search)

为对话里的 `web_search` 工具配置搜索服务商。

**入口**:设置 → **网页搜索** 面板。9 家服务商可拖拽排序(排第一的是主用,其余启用的作兜底):

- **免费无需 Key**:DuckDuckGo(标注「可靠性有限」)
- **自托管**:SearXNG(填实例 URL,含 Docker 一键部署)
- **需 Key**:Tavily(全球推荐)、Bocha 博查(国内推荐)、Brave、Perplexity、Google CSE、Grok、Kimi

| 高级设置 | 作用 |
| --- | --- |
| 默认结果数 | 每次返回条数(1–10) |
| 超时 | 5–120 秒 |
| 缓存 TTL | 0–60 分钟 |
| 默认国家 / 语言 / 时效 | 搜索的地域、语种、新鲜度过滤 |

没有总开关,靠「是否有启用的服务商」决定是否可用。服务商 Key 可以让 AI 通过对话配置,但读取时会打码。

### 网页抓取(web_fetch)

为 `web_fetch` 工具配置抓取参数(纯参数,无需服务商)。

**入口**:设置 → **网页抓取** 面板。

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 最大字符数 | 50000 | 注入模型的内容上限 |
| 最大响应体 | 2 MiB | 下载体积上限 |
| 超时 | 30 秒 | 1–120 秒 |
| 最大重定向 | 5 | 0–20 |
| 缓存 TTL | 15 分钟 | 0–1440 分钟 |
| **SSRF 防护** | **开** | 拦截访问内网 / 元数据地址,防止服务端请求伪造 |

> **建议保持 SSRF 防护开启**。关闭会放行内网地址,存在安全风险。所有出站请求都遵循统一的安全策略,详见 [13 · 设置与安全](13-设置与安全.md)。
