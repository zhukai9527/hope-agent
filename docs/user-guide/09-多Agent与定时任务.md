## 9.1 子 Agent

主 Agent 可以把一个独立子任务派给另一个(或同一个)Agent 在**隔离会话**里异步执行,完成后结果自动回流到当前对话,让主 Agent 接着往下做。这一切由 AI 自主发起,你不用手动操作,但可以在设置里管控。

- **前台等待**:默认等 30 秒,超时未完成会自动转入后台继续跑,不阻塞你的对话。
- **批量派发**:一次并行派多个子任务,全部完成后合并成一条汇总一次性注入(省一轮成本)。
- 运行中可以引导(推送提示)、取消单个或全部。
- 子 Agent 默认在独立的[隔离工作树](07-工具与权限.md#79-文件操作git-与隔离工作树)里跑,不污染你的主工作区。

**查看运行情况**:对话里每个子 Agent 显示为一枚胶囊(头像、名称、状态、耗时),点击打开右侧「子智能体」面板。

- 列表按**运行中 / 已完成**分组,涵盖本会话全部子 Agent——前台、后台,以及 Workflow 派生的都在内。
- 点进某一个,可以看它的**结果**、**调用信息**(服务商、模型、思考强度、起止时间、附件、嵌套深度等)和**实时对话**;已完成的默认落在结果,运行中的默认落在对话。
- 子 Agent 自己再派的下一层可以继续点进去,用面包屑逐级返回。
- 工作台里也有「子智能体」区块,列出同一份清单。

**设置**(在 Agent 编辑页 → 子 Agent 调用 标签):

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 启用子 Agent | 开 | 该 Agent 是否可派子 Agent |
| 允许的子 Agent | 全部 | 勾选哪些 Agent 可被当子 Agent |
| 最大嵌套深度 | 3 | 子 Agent 再派子 Agent 的层数(1–5) |
| 最大并发 | 8 | 单会话同时运行的子 Agent 上限 |
| 默认超时 | 0(不超时) | 子 Agent 执行超时秒数 |

> 你新发的消息优先级高于自动注入——一发消息就会取消在途注入,注入排队到空闲后重试(不丢结果)。后台子 Agent 会投影进统一的[后台任务面板](#93-后台任务面板)。

## 9.2 Agent 团队

团队让多个具名 Agent 作为成员**并行协作**——成员之间可以互发消息、共享一块看板任务,由协调者统一编排。这区别于子 Agent「父派子、子回收」的单向模式。

**怎么用**:

- 由 AI 调用 `team` 工具创建(用团队模板一键铺开成员),或用 `/team` 命令管理。
- **模板在设置里预配**:设置 → 团队,添加 / 编辑模板,每个模板定义成员的名字、绑定的 Agent 和角色。
- 团队运行时右侧会弹出团队面板,有三个标签:仪表板(成员卡片 + 进度 + Token 统计)、任务(待办 / 进行中 / 审查 / 完成四列)、消息(实时消息流,可手动给团队发消息)。

**设置**(在 Agent 配置里):是否允许创建团队、最大活跃团队数(默认 3)、每团队最大成员数(默认 8)、成员默认模型。

> 目前**没有内置团队模板**,全部由你在设置里自定义。模板引用的 Agent 被删除后,创建团队时会报错。

## 9.3 后台任务面板

所有异步 / 后台执行的工具、子 Agent、批量任务统一进一张面板,可以实时看状态、看运行输出、随时取消。

- **入口**:聊天头部的徽标、独立面板,以及工作台里的后台任务区块。
- 后台执行命令时可以实时看到输出「尾巴」,判断是「还在跑」还是「卡住了」。
- 任务完成后可以发桌面通知(默认开)。

哪些工具能后台运行:执行命令、浏览器、网络搜索、AI 画图、音频生成、应用更新。

**设置(设置 → 工具设置 → 异步工具,中风险)**:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 启用异步工具 | 开 | 后台化总开关 |
| 自动转后台阈值 | 0(关) | 同步工具跑超过此秒数自动转后台 |
| 完成合并窗口 | 3 秒 | 同会话多个任务在窗口内完成合并为一轮注入(省成本) |
| 全局并发上限 | 8(按核数) | 所有会话的后台任务并发数 |
| 每会话并发上限 | 6(按核数派生,恒小于全局) | 单会话后台任务并发数,超出排队 |
| 失败自动重试 | 关 | 仅对无副作用的工具(如网络搜索)重试;执行命令 / 画图等永不重试 |

## 9.4 定时任务(Cron)

让 AI 按计划自动跑一段对话,结果可以发桌面通知,或投递到指定的 IM 会话。

### 创建方式

- **自然语言**:直接对 AI 说「每天早上 9 点提醒我喝水」,它会帮你创建。
- **表单**:定时任务面板 → 新建,填名称、调度、消息、Agent、投递目标等。

### 调度类型与时区

| 类型 | 说明 |
| --- | --- |
| 一次性(At) | 指定日期时间触发一次 |
| 固定间隔(Every) | 每 N 分 / 时 / 天触发(下限 1 分钟) |
| cron 表达式(Cron) | 标准 cron,GUI 有可视化构建器;带 **IANA 时区选择器**,默认填你的浏览器时区,按该时区的墙上时钟解释、支持夏令时 |

### 把结果发到 IM

任务结果的最终文本可以发到一个或多个 IM 会话——先选渠道账号再选会话。

- **投递白名单**:目标必须是已记录的真实 IM 会话,防止被注入的 AI 把周期任务变成外泄通道。
- **前缀开关**(默认关):成功投递时加 `[Cron] 任务名` 前缀,方便区分投到同一群的多个任务。
- 账号被删导致目标失效时,界面会标红提示,投递时跳过。

### per-job 设置(仅所有者可设)

| 设置项 | 默认 | 说明 |
| --- | --- | --- |
| 权限模式覆盖 | 跟随 Agent | 该任务用标准 / 智能 / YOLO;**只能在界面里设,AI 工具不能改**(防提权) |
| 沙箱模式覆盖 | 跟随 Agent | 选非「关闭」需要 Docker,不可用则终止本次运行、绝不裸跑 |
| 超时覆盖 | 用全局 | 让长任务声明自己的预算,而不用抬高全局值 |
| 最大失败次数 | 5 | 连续失败达此值自动禁用(0 = 永不自动禁用) |

任务连续失败达到上限会**自动禁用并弹专属通知**;基础设施类失败(会话没起来)不计入。

### 集中展示与未读

每次运行都新建一个隔离会话(标题就是任务名),**不再混入主侧边栏会话列表**。你可以在定时任务面板的「历史」视图里集中查看:左栏是跨任务的运行时间线,右栏是该次运行的完整对话(只读)。侧边栏的定时任务图标有独立的未读角标,可一键「全部已读」。定时任务的未读与普通对话独立,不计入 Dock / 托盘。

### 全局设置(设置 → 定时任务,中风险)

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 最大并发 | 5 | 调度器全局并发上限(0 = 不限) |
| 全局超时 | 0 | 每次运行的时间预算(0 = 不加超时) |
| 补跑窗口 | 300 秒 | 一次性任务错过后的补跑窗口(0 = 严格不补跑) |

## 9.5 自我唤醒(schedule_wakeup)

这是 AI 的一次性「N 秒后把我叫回当前会话续跑」的能力,和定时任务不同:

| 维度 | 自我唤醒 | 定时任务 |
| --- | --- | --- |
| 语义 | AI 主动「过一会儿叫我回来」 | 独立的计划任务 |
| 次数 | **一次性** | 一次或循环 |
| 回到哪 | 当前会话 | 每次新建隔离会话 |
| 适用 | 短周期临时等待(等某个任务跑完) | 长周期 / 循环 / 需要投递的任务 |

延迟范围 10 秒到 24 小时(默认上限,可在设置里最高调到 7 天),每个会话最多挂 5 个待唤醒。
