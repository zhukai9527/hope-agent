## Hope Agent 是什么

Hope Agent 是一个**本地优先、桌面优先,也能服务化常驻**的个人 AI 助手。它像成熟的桌面软件一样打开即用,又真正具备 Agent 的行动力:

- **会记忆**——跨会话记住你的偏好、项目背景与长期习惯,越用越懂你;
- **能成长**——把复杂任务沉淀成可复用的技能,把资料沉淀成第二大脑;
- **能自主推进目标**——你给出目标,它在后台持续拆解、执行、检查、推进;
- **真正能操作**——在你的授权与审批下运行命令、读写文件、控制浏览器和电脑、调用外部工具;
- **随叫随到**——桌面、浏览器、常用 IM(Telegram / 飞书 / Slack 等)之间共享同一套会话、记忆与配置。

数据默认全部保存在你自己的电脑(`~/.hope-agent/`),模型请求直连你配置的 Provider,不经任何第三方中转。

## 核心概念(一次讲清)

理解下面这几组概念,能帮你快速看懂全书:

**三种运行模式**——同一套核心,三个入口:
- **桌面 GUI**:功能最全的原生应用(macOS 完整支持,Windows / Linux 为实验性),打开即用。
- **Server + Web GUI**:`hope-agent server` 常驻后台(NAS / 服务器 / 云主机),浏览器打开即得完整网页版,手机平板都能连。
- **ACP**:`hope-agent acp`,把 Hope Agent 当作 IDE 的 AI 后端。

**自主执行的五个角色**(可组合、也可单独用):
- **目标(Goal)**——定义「最终要达成什么、怎样算完成」。
- **工作流(Workflow)**——负责「一次具体的执行」,可含阶段、并行、多 Agent、审阅与验证。
- **持续推进(Loop)**——决定「何时再次推进」(定时、条件或模型自定时间)。
- **任务(Task)**——呈现「当前进度」。
- **执行模式(Mode)**——控制「自主执行的强度」。

**三个专属空间**:
- **记忆系统**——跨会话记住重要事实,分全局 / 项目 / Agent 三层,按需召回。
- **知识空间**——你和 AI 共同读写的真实 Markdown 笔记库,你的「第二大脑」。
- **设计空间**——从想法到可交付设计产物的创作空间。

**工具与审批**——AI 通过「工具」实际操作你的电脑、文件、浏览器和外部服务;敏感操作会进入**审批**流程,你可以选择每次确认、始终允许或拒绝,高风险命令还能放进 **Docker 沙箱**执行。权限有三档模式:`默认`、`智能`、`YOLO(完全放行)`。

**本地优先**——配置、会话、记忆、附件、技能、日志默认都在本机 `~/.hope-agent/`;API Key 直连模型 Provider;服务模式提供 Bearer Token 鉴权与 SSRF 防护。
