# 07 · 工具与权限

Hope Agent 之所以「能干活」,是因为 AI 可以调用**工具**去实际操作:执行命令、读写文件、搜索网络、控制浏览器和电脑。为了让你放心把这些能力交给它,所有敏感操作都会经过**审批**。本章讲清楚有哪些工具、怎么管控权限、以及沙箱、浏览器控制和电脑控制。

## 7.1 内置工具箱一览

AI 会**自动**在需要时调用这些工具,你通常不用手动触发,但可以在权限设置里管控它们。下面是主要工具(按类别):

| 类别 | 工具 | 做什么 |
| --- | --- | --- |
| **命令** | `exec` | 执行 shell 命令(有独立的命令级审批,支持后台运行、超时、沙箱) |
| | `process` | 管理后台运行的命令(查看输出、写入、结束) |
| **文件** | `read` / `write` / `edit` / `apply_patch` | 读取 / 写入 / 精确修改 / 批量补丁(写类操作需审批) |
| | `ls` / `grep` / `find` | 列目录 / 搜索内容 / 按模式找文件 |
| | `lsp` | 语义代码智能(定义 / 引用 / 诊断等) |
| **网络** | `web_search` / `web_fetch` | 网络搜索 / 抓取网页正文 |
| **多模态** | `image` / `pdf` | 让模型看图片 / 解析 PDF |
| | `image_generate` | AI 画图 |
| | `audio_generate` | AI 生成语音 / 音乐 / 音效 |
| **浏览器 / 电脑** | `browser` | 驱动浏览器(见 [7.7](#77-浏览器控制)) |
| | `mac_control` | 控制 macOS 桌面(见 [7.8](#78-电脑控制macos)) |
| **记忆 / 知识** | 记忆与笔记工具 | 保存 / 检索记忆、读写笔记(内部调用,不弹审批) |
| **协作** | `subagent` / `team` / `acp_spawn` | 调用子 Agent / 团队 / 外部 ACP Agent |
| **控制面** | `task_*` / `goal_*` / `workflow` / `loop_*` | 任务进度、目标、工作流、持续推进 |
| **交互** | `ask_user_question` | 向你提结构化问题 |
| | `send_notification` / `send_attachment` | 发系统通知 / 推送可下载文件 |
| **其它** | `canvas` / `get_weather` / `manage_cron` / `schedule_wakeup` 等 | 可视化沙盒 / 天气 / 定时任务 / 自我唤醒 |

**工具通用设置(设置 → 工具设置 → 通用)**:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 工具超时 | 0(不限) | 单次工具执行的超时秒数 |
| 工具结果落盘阈值 | 50 KB | 超过则写盘,上下文只留头尾预览 |
| 延迟工具加载 | 推荐(默认开) | 默认延迟一批低频 / 大 schema 工具,按需经 tool_search 发现(省上下文) |
| 最多图片 / PDF / 视觉页数 | 10 / 5 / 10 | 单次上限 |

## 7.2 权限模式三选一

每个会话独立选一个权限模式,决定工具调用要不要弹审批。**切换方式**:聊天输入区工具栏的权限模式切换器、`/permission` 命令,或 `Shift+Tab` 快速循环。

| 模式 | 行为 | 适合谁 |
| --- | --- | --- |
| **默认(default)** | 编辑类操作强制审批,加上你为 Agent 自定义勾选的工具 | 大多数用户 |
| **智能(smart)** | 由 AI 判断:高度确信的自动放行,或用一个「判官模型」来决定;已经编辑过的文件再改自动放行 | 进阶用户,信任 AI 判断 |
| **完全放行(yolo)** | 该会话内全部放行(只有计划模式还能拦) | 一次性脚本、极度信任的场景 |

**关于智能模式**:它有三种策略——只看 AI 自报的确信度、直接用判官模型、或两者结合。**智能模式会忽略「自定义工具审批」的勾选**(界面会提示你)。但保护路径、危险命令、浏览器底层命令、外部服务写操作等,在智能判断之前就已经被拦住,不受影响。

> **计划模式优先级最高**——即使在全自动模式下,[计划模式](08-自主任务.md#85-计划模式plan-mode)也能拦住不该做的事。

## 7.3 审批弹窗

当工具需要确认时,会弹出审批对话框,有三个选择:

- **拒绝(Deny)**
- **仅本次允许(Allow Once)**
- **始终允许(Allow Always)**——记住这个决定,以后同类不再问

弹窗里会显示操作原因、工作目录、命令 / 操作摘要,还有一个倒计时圆环(如果开了[审批超时](#75-全自动超时与无人值守))。

> **有些操作不能「始终允许」**(每次都要手动确认)——包括保护路径、危险命令、高风险的 macOS 控制、浏览器底层命令、外部服务的写操作、以及计划模式下的提问。这些是最敏感的操作,刻意不给你「一劳永逸放行」的选项。
>
> **多端一致**:同一个审批可以同时出现在桌面、网页和 IM 上,任一端处理后其它端会统一撤掉并提示「已由其它端处理」。

## 7.4 保护路径、危险命令与编辑命令

这是三张可编辑的列表,命中就强制审批。**入口**:设置 → 权限。

| 列表 | 触发 | 例子 | 能否「始终允许」 |
| --- | --- | --- | --- |
| **保护路径** | 读 / 写 / 编辑 / 执行 | `~/.ssh/`、`~/.aws/`、`/etc/`、`.env`、`*secret*`、`*.pem`、`*.key` | 否 |
| **危险命令** | 执行命令 | `rm -rf /`、`git push --force`、`git reset --hard`、`mkfs`、`DROP TABLE` | 否 |
| **编辑命令** | 执行命令 | `rm `、`mv `、`git commit`、`npm install`、`> `、`>>` | 可(仅默认模式生效) |

你可以在这三张列表里增删条目,也可以「恢复默认」。这些设置属于**高风险**(权限规则)。

> 此外,飞书写工具和外部连接器(Gmail / Calendar / Drive / Slack / Notion 等)的写操作会被自动识别为最严格的审批,不进这些列表,也不能「始终允许」。

## 7.5 全自动、超时与无人值守

### 全局 YOLO(全局跳过审批)

一键跳过**全部**工具审批,极高风险,只在完全信任的本地环境用。

- **入口**:设置 → 安全策略 → 危险模式(开启需勾选「我已了解风险」),或命令行启动加 `--dangerously-skip-all-approvals`。
- 开启后,执行命令、写 / 编辑文件、浏览器操作、所有 IM 触发的工具调用都会放行;命中保护路径 / 危险命令时只记审计日志、不弹窗。计划模式的限制仍然生效。

### 审批超时

**默认永不超时**——审批会一直等你。你可以在设置 → 权限里开启超时:

| 设置项 | 默认值 | 作用 |
| --- | --- | --- |
| 自动超时 | 关(永不超时) | 是否启用超时 |
| 超时秒数 | 300 秒 | 等待时长(0 = 无限) |
| 超时动作 | 拒绝 | 到时:拒绝 / 放行 |

> **重要**:「超时放行」只对普通操作生效;最敏感的操作(保护路径、危险命令等)即使设了放行,**超时也永远强制拒绝**。

### 无人值守

当工具需要审批但确实没人能回应时(比如凌晨自动跑的定时任务、没有界面的服务器),默认按**拒绝**处理(设置 → 权限 → 无人值守)。只要有任何可能有人在(桌面窗口、网页客户端、IM),就当作有人值守正常弹窗——只有定时任务这类确证无人的场景才走无人值守策略。

## 7.6 Docker 沙箱

沙箱让 `exec` 命令在 Docker 容器里跑,而不是直接在你电脑上执行,保护宿主机。

**怎么用**:聊天输入区权限切换弹层里的「沙箱」分区(会话级),或在 Agent 设置里设默认沙箱模式。

| 模式 | 执行位置 | 文件写入 |
| --- | --- | --- |
| 关闭(默认) | 你的电脑 | 真实生效 |
| 标准 | Docker,挂载当前目录 | 落真实工作区 |
| 隔离 | Docker + 临时副本 | 只落临时副本,结束删除 |
| 工作区 | Docker 挂载工作区 | 落真实工作区(工作区内命令审批放松) |
| 信任 | Docker 内最大自治 | 同工作区(最敏感操作仍每次审批) |

> **选了沙箱但 Docker 不可用时,工具执行会直接失败(报错),绝不会悄悄退回到你的电脑上跑**。沙箱不会绕过权限——最敏感的操作照样每次审批。

**容器配置(设置 → Docker 沙箱)**:镜像(默认 `debian:bookworm-slim`)、内存(512 MB)、CPU(1 核)、只读根文件系统(开)、网络模式(默认 none 无网络)等。面板顶部会显示 Docker 是否可用,未装 / 未运行时按平台给安装引导。

## 7.7 浏览器控制

AI 通过 `browser` 工具驱动浏览器,可以打开网页、点击、填表、抓取内容、截图等。

**两种后端**:

- **Chrome 扩展(默认)**——通过一个 Chrome 扩展控制你**真实登录的 Chrome**,带上你所有的 cookie 和登录态,能直接操作已打开的标签页。
- **独立浏览器(降级 / Docker 用)**——用一个独立的 Chromium 实例,与你的真实 Chrome 数据隔离。

后端偏好可在设置 → 浏览器里选(`扩展优先` / `仅 CDP` / `仅扩展`)。

**实时镜像**(桌面独占):聊天右侧有一个浏览器面板,实时镜像 AI 正在操作的页面,你能看到它在访问和点什么。

**安全**:

- 发送**原始 DevTools 命令**(底层浏览器命令)是最严格的操作——**每次都审批、永远不能「始终允许」**;还有一个硬开关可以彻底禁止 AI 发送这类命令。
- 执行任意 JavaScript、接管你的真实标签页、读真实下载等,都会弹审批。
- 所有导航的网址都会过安全检查(防止访问内网)。
- 遇到登录、二次验证、验证码,AI 一律通过提问交给你处理。

## 7.8 电脑控制(macOS)

> 仅 **macOS 桌面版**真正可用。网页 / 服务器模式和其它平台不支持。

AI 通过 `mac_control` 工具读取屏幕和无障碍信息,操作应用、窗口、菜单、剪贴板和对话框。

**先授权**(设置 → 系统权限,仅 macOS):

- **辅助功能**——读取界面元素、点击、输入、控制窗口和菜单;
- **屏幕录制**——截图、右侧镜像面板、视觉定位。

面板会显示当前授权状态和缺失的权限,点「授权」弹系统弹窗、点「检查」跳系统设置。

**能做什么**:只读类(查看前台应用 / 窗口 / 元素树、截图给模型看、找元素、列应用 / 窗口 / 菜单)一般直接放行;操作类(点击、输入、粘贴、快捷键、拖拽、切换应用 / 窗口 / 桌面、点菜单、读写剪贴板)需要审批;**高风险类**(退出应用、关闭窗口、点「删除 / 清空 / 重置」这类危险按钮)不能「始终允许」,每次都要确认。

> AI 不读密码字段的真实内容,不把截图塞进上下文,也不能后台控制 Hope Agent 自己的窗口。审批弹窗前会记住当前前台应用、审批后恢复,避免抢焦点。

## 7.9 文件操作、Git 与隔离工作树

### 文件操作

文件的打开 / 下载 / 预览由**文件所在的机器**决定:

- **本地桌面**——用系统默认应用打开,可在文件管理器中定位。
- **网页 / 远程**——在浏览器 / 应用内下载。

可预览的类型(代码、文本、Markdown、图片、PDF、音视频、Office 文档)会在右侧预览面板里直接看。文件在界面里可编辑(有效文本、5 MB 以内),保存时有冲突检测(只提供重载 / 另存 / 取消,不会强制覆盖)。

> 网页 / 服务器模式下,对服务器文件的写操作受「允许远程写入」开关约束(默认关);桌面本机不受限。

### Git 控制

当会话在一个 Git 仓库里时,工作台会出现 Git 控制卡:审阅 Diff(可按文件 / 按 hunk 暂存、撤销)、切换 / 创建分支、暂存提交、推送、以及创建和管理 GitHub PR(检查状态、评审、合并)。

> Git 控制**不会自动 fetch / stash / pull / rebase,也不提供强制推送**——避免意外覆盖你的工作。PR 里的外部文本被当作不可信数据处理。

### 隔离工作树(Worktree)

长任务、[工作流](08-自主任务.md#82-工作流workflow)和[子 Agent](09-多Agent与定时任务.md#91-子-agent)可以在一个**独立目录**里改代码,不污染你的主工作区。创建工作流时可以选「在哪运行」(当前目录 / 新隔离工作树 / 已有工作树);用户委派的子 Agent 默认会尝试建隔离工作树。本地和工作树之间的迁移会完整复制改动并做校验,失败自动回滚。
